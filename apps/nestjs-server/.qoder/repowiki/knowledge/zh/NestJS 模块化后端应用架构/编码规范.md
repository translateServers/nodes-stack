- 所有业务模块遵循 Controller-Service-Module 分层结构，DTO 使用 Zod Schema 定义并配合 nestjs-zod 进行自动验证。
- 全局异常通过 HttpExceptionFilter 统一捕获并转换为标准化的 API 响应格式（包含 code, message, data）。
- 配置项通过 src/config/schemas 下的 Zod Schema 定义，并由 TypedConfigService 提供类型安全的访问接口。
- 数据库模型定义在 prisma/schema 目录下，通过 Prisma Client 生成类型安全的查询接口，并在 Service 层注入使用。
