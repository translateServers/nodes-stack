- 采用 NestJS 模块化架构，通过 `AppModule` 聚合基础设施模块（Prisma、Cache、Logger、Config）与核心业务模块（Auth、User、Health）。
- 全局拦截器（TransformInterceptor、LoggingInterceptor）、守卫（JwtAuthGuard、ThrottlerGuard）和过滤器（HttpExceptionFilter）在根模块统一注册，确保跨模块行为一致性。
- 使用 `TypedConfigService` 结合 Zod Schema 实现类型安全的配置管理，并在 `main.ts` 中统一初始化 Swagger 文档、CORS 策略及全局前缀。
- 依赖注入机制贯穿全栈，业务模块通过构造函数注入服务与仓储，基础设施模块通过导出 Provider 供业务层复用。
