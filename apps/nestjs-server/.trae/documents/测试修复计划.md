# 测试修复计划

## 概述

架构改进实施完成后，lint 和 typecheck 均通过，但 8 个测试用例失败，分布在 2 个测试文件中。本计划修复所有失败测试。

## 当前状态分析

### 失败测试 1: `auth.service.spec.ts`（3 个失败）

**根因**: Mock `TypedConfigService` 是一个普通对象 `{ jwtSecret: 'test-secret', ... }`，没有 `get()` 方法。而 `AuthService` 通过 `this.config.get('jwt.secret')` 等点语法路径访问配置。

**影响范围**:

- `loginWithCredentials` - 调用 `generateTokens` 时触发 `this.config.get()`
- `register` - 同上
- `refreshToken` - 同上，且 `findUnique` 现在使用 SHA-256 哈希值查询

**具体问题**:

1. `mockEnvConfig` 对象无 `get` 方法 → `this.config.get is not a function`
2. `refreshToken` 测试中 `findUnique` 的 `where: { token }` 应为哈希值而非明文

### 失败测试 2: `user.service.spec.ts`（5 个失败）

**根因**: `UserService` 的 `create`/`findAll`/`findOne`/`update` 方法返回 `UserResponseSchema.parse(result)`，该 schema 使用 `DateTimeStringSchema` 将 `Date` 对象转换为 `"YYYY-MM-DD HH:mm:ss"` 字符串。但测试中 mock 数据使用 `new Date()` 作为日期字段，断言 `expect(result).toEqual(mockUser)` 时类型不匹配。

**影响范围**:

- `create` - `expect(result).toEqual(mockUser)` 中 mockUser 的 createdAt/updatedAt 是 Date，result 的是 string
- `findAll` - 同上
- `findOne` - 同上
- `update` - 同上
- `remove` - 方法返回 `Promise<void>`，但测试期望返回用户对象

## 修改方案

### 文件 1: `src/modules/auth/auth.service.spec.ts`

**修改 mock TypedConfigService**:

- 将 `mockEnvConfig` 从普通对象改为带 `get` 方法的 mock 对象
- `get` 方法支持点语法路径解析（与 `TypedConfigService.get()` 行为一致）

```typescript
const mockConfigData = {
  jwt: {
    secret: 'test-secret-that-is-at-least-32-characters-long',
    accessTokenTtl: '15m',
    refreshTokenTtl: '7d',
    refreshSecret: 'test-refresh-secret-that-is-at-least-32-chars',
  },
};

const mockTypedConfigService = {
  get: jest.fn((path: string) => {
    const keys = path.split('.');
    let result: unknown = mockConfigData;
    for (const key of keys) {
      result = (result as Record<string, unknown>)[key];
    }
    return result;
  }),
};
```

**修改 `refreshToken` 测试**:

- `findUnique` 的 `where: { token }` 应期望 SHA-256 哈希值
- 使用 `createHash('sha256').update(oldRefreshToken).digest('hex')` 计算期望的哈希值

### 文件 2: `src/modules/user/user.service.spec.ts`

**修改 mock 数据中的日期字段**:

- 将所有 `createdAt: new Date()` / `updatedAt: new Date()` 改为 `dayjs().format('YYYY-MM-DD HH:mm:ss')` 格式的字符串
- 这样 `UserResponseSchema.parse()` 的输出与 mock 数据一致

**修改 `remove` 测试**:

- `remove()` 方法返回 `Promise<void>`，测试应验证方法调用而非返回值
- 移除 `expect(result).toEqual(existingUser)` 断言
- 改为验证 `findUnique` 和 `delete` 被正确调用
