## 基础规则

1. 使用pnpm作为包管理器
2. 禁止启动运行项目
3. 禁止滥用any与as
4. 项目使用prisma7作为数据库操作工具

## 编码思维模式

每次编写代码前，按以下顺序思考：

```
1. 我需要引入什么？ → 立即标记为"待使用"，写完后检查是否全部使用
2. 这个函数的参数/返回值类型是什么？ → 先定义类型再写逻辑
3. 这里可能为 null/undefined 吗？ → 立即处理，不要等报错
4. 这个 Promise 怎么处理？ → 立即决定 await 还是 .catch()
5. 写完后检查：有未使用的导入吗？有any/类型断言吗？格式符合Prettier吗？
```

## 🚫 红线规则 (绝不允许)

- `/* eslint-disable */` 或 `// eslint-disable-next-line`
- `@ts-ignore` 或 `@ts-expect-error`
- 未使用的 import (写完就删掉不用的)
- 声明但从未使用的变量

## 代码质量规范

详细的 Prettier 格式规则、TypeScript 类型规范、ESLint 自检清单请调用 `typescript-quality` Skill 获取。

## 验证 (最后保障)

代码生成后运行以下命令做最终确认:

```bash
pnpm run lint && pnpm run typecheck && pnpm test
```
