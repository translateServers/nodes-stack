# Repowiki 维基文档应用规范

## 文档体系结构

项目使用 `.qoder/repowiki/` 目录管理维基文档，包含两套体系：

- **knowledge/**：知识卡模块，按模块组织，每个模块包含 `_module.yaml` 和标准文档文件
- **zh/content/**：中文内容文档，按主题分类组织，包含详细的项目文档

## 当前文档覆盖范围

repowiki 已收录项目全部内容，涵盖以下主题：

- **knowledge/zh/Nebula 全栈 Monorepo 架构/**：项目整体知识卡（概述、技术栈、架构设计、编码规范、特殊配置）
- **zh/content/项目概述/**：项目总览与快速开始
- **zh/content/后端系统/**：NestJS 架构设计、认证授权、用户管理、健康监控、配置管理、日志记录、数据访问层、中间件与拦截器
- **zh/content/前端系统/**：React 应用架构、路由导航、开发工具链、API 集成、UI 组件库、状态管理系统
- **zh/content/共享包/**：共享包、共享类型定义、共享工具函数、ESLint/TypeScript 配置包
- **zh/content/架构设计/**：Monorepo 架构、前端架构、后端架构、安全架构、数据流设计
- **zh/content/API 文档/**：Swagger、认证 API、用户管理 API、字典管理 API、菜单权限 API、健康监控 API、系统健康 API
- **zh/content/数据库设计/**：Prisma Schema 与数据模型
- **zh/content/测试策略/**：单元测试、集成测试、测试配置
- **zh/content/开发工具/**：IDE 配置、代码格式化、代码检查、构建系统
- **zh/content/部署指南/**：容器化部署、编排部署、CI/CD 流水线、环境配置、监控告警
- **zh/content/故障排除/**：开发环境问题、运行时错误、数据库问题、性能问题、日志和调试

## knowledge 模块规范

### 模块目录结构

每个模块目录必须包含以下标准文件：

```
knowledge/zh/{模块名}/
├── _module.yaml    # 模块定义文件
├── 概述.md          # 模块概述（必需）
├── 技术栈.md        # 技术栈说明（必需）
├── 架构设计.md      # 架构设计说明（必需）
└── 编码规范.md      # 编码规范（必需）
```

### \_module.yaml 规范

- 必须声明 `schema_version`、`module_path`、`title`、`scope`
- `scope` 列出模块关联的源文件路径，使用相对路径
- `depends_on` 和 `related_to` 声明模块间依赖和关联关系

### \_index.yaml 规范

- 根索引文件声明 `schema_version`、`locale`、`branch`、`exported_at`
- `modules` 下按模块 key 列出所有模块，包含 `dir_name`、`title`、`scope`

## content 文档规范

### 目录组织

- 按主题分类创建子目录，如：`项目概述/`、`架构设计/`、`API 文档/`
- 每个子目录包含一个同名的总览文件和若干细分文档
- 支持多层嵌套，如 `架构设计/前端架构/状态管理系统/`

### 文档格式要求

- 使用 Markdown 格式，文件名使用中文
- 每篇文档必须包含一级标题（`#`）
- 引用源文件时使用 `<cite>` 块标注引用的文件路径
- 架构和流程说明优先使用 Mermaid 图表（`graph`、`sequenceDiagram`、`flowchart`）
- 图表和关键结论后必须标注章节来源（文件路径和行号）

### 文档内容标准

- **概述类**：简明扼要描述模块/系统定位与核心能力，1-2 句话
- **技术栈类**：列出关键技术及版本，逗号分隔
- **架构设计类**：包含架构图、核心流程图、关键设计决策说明
- **编码规范类**：列出该模块的编码约束和最佳实践
- **API 文档类**：包含请求参数、响应格式、状态码说明

## 新增文档流程

1. 确定文档所属模块和分类目录
2. 在 `knowledge/zh/` 下创建或更新模块目录及 `_module.yaml`
3. 在 `zh/content/` 对应分类目录下创建 Markdown 文档
4. 更新 `_index.yaml` 索引，将新文档路径加入对应模块的 `scope`
5. 确保文档中引用的源文件路径正确且文件存在

## 维护注意事项

- 新增或重构源文件后，检查对应 repowiki 文档是否需要同步更新
- `scope` 中的文件路径必须与实际文件一致，定期清理已删除的路径
- 避免将构建产物（`.turbo/cache/`）、锁文件（`pnpm-lock.yaml`）、数据库文件（`dev.db`）等纳入 `scope`，这些不应出现在知识卡索引中
