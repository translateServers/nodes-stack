# Tasks

## 执行原则

- 本文件只拆分阶段 2 实施任务，不代表产品代码已经开始或任务已经完成。
- 阶段 1 checklist、验证记录和关键回归全部有效是强制门禁；门禁失效时不得开始或继续阶段 2。
- 严格采用"共享契约 → 纯函数解析 → 静态闭环 → API 闭环 → 状态与刷新 → 历史扩展 → 脱敏 → 端到端验证"的顺序。
- 每个任务只交付一个可观察、可验证、可回滚的结果；不得一次性重写属性面板或编辑器 Store。
- 保留手写 SVG renderer、现有 Moveable/Selecto 交互链路和属性面板骨架；不引入 ECharts/Recharts 等图表库。
- Schema 定义与接入调用在存在风险时拆成两个任务；先共享包契约，再前端消费，再服务端脱敏。
- 取消、无变化和失败操作不得产生空历史记录或错误脏状态。
- 不在阶段 2 引入 POST 写请求、服务端代理、图表库、其他组件类型数据源、服务端修订版、角色工作区或多人协作。
- 任务状态仅在实现、自动化验证和必要浏览器证据真实完成后从 `[ ]` 改为 `[x]`。
