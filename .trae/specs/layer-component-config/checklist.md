# Checklist

本 checklist 用于验收 `layer-component-config` 阶段 2 实现。所有实施条目默认未完成；只有产品行为、自动化验证和必要浏览器证据均存在时才能勾选。

## 规格边界

- [ ] 实施范围只覆盖四层配置契约、数据解析器、bar-chart 数据源闭环、数据源配置 UI、三态、定时刷新、画布配置历史和敏感请求头保护。
- [ ] 未引入 POST、PUT、DELETE 等写请求方法或请求体编辑。
- [ ] 未引入 WebSocket 或其他数据源类型。
- [ ] 未引入 ECharts、Recharts 等图表库；bar-chart 仍为手写 SVG renderer。
- [ ] 未引入多系列、聚合、分组、下钻或图表联动能力。
- [ ] 未将数据源接入扩展到 bar-chart 以外的组件类型。
- [ ] 未建设服务端 API 代理、跨域转发或凭证托管。
- [ ] 未引入全局共享数据源管理。
- [ ] 未引入服务端修订版（revision）、项目分支、实时多人同步或 AI 推荐。
- [ ] 未引入角色工作区或按角色的面板裁剪。
- [ ] 未为 bar-chart 以外的组件类型建立 props 强类型契约。
- [ ] 未一次性重写属性面板或编辑器 Store。

## 阶段进入门禁

- [ ] 阶段 1 checklist 已全部完成。
- [ ] 阶段 1 验证记录存在且可追溯。
- [ ] 阶段 1 关键回归当前有效。
- [ ] 任一阶段 1 门禁失效时阶段 2 已停止并先恢复基线。

## 实施基线

- [ ] 已记录 bar-chart `props.data` 直塞、`dataSource` 未消费、`updateCanvas` 不入历史、JSON textarea 编辑和无解析器的实施前事实。
- [ ] 已执行实施前前端 screen 定向测试并记录实际结果。
- [ ] 已执行实施前后端 screen 定向测试并记录实际结果。
- [ ] 已执行实施前 screen Playwright E2E 并记录实际结果。
- [ ] 已执行实施前 typecheck、lint 和 Biome 并记录实际结果。
- [ ] 数据源配置 UI 与三态展示使用稳定、可访问的定位契约。
- [ ] API 数据源 E2E 使用 route Mock，不依赖真实外部网络。

## 四层配置契约

- [ ] 数据层配置只存在于 `dataSource` 字段。
- [ ] 数据层包含数据源类型、静态数据、API 配置、可选数据路径与可选字段映射。
- [ ] 逻辑层配置只存在于 `logic` 字段，包含排序与条数限制。
- [ ] 视觉层配置只存在于 `style` 与视觉类 props。
- [ ] 交互层配置只存在于 `interaction` 字段。
- [ ] `position`、`zIndex`、`status`、`parentId` 未被混入四层配置。
- [ ] 每一层配置可单独通过共享 Zod Schema 校验。
- [ ] 单层校验错误不掩盖其他层校验结果。
- [ ] 校验错误信息可定位到具体层与字段。
- [ ] 修改任一层时其余三层配置值不变。
- [ ] 服务端保存接口使用同一共享 Schema 校验四层配置。
- [ ] 非法四层配置不会写入数据库。

## 向后兼容

- [ ] 仅有 `props.data` 的旧 bar-chart 可解析、可渲染。
- [ ] 无 `logic`、`interaction` 字段的旧项目数据可解析。
- [ ] 旧组件视觉表现与契约演进前一致。
- [ ] 首次通过数据层 UI 提交时 `props.data` 一次性迁移为数据层静态数据。
- [ ] 迁移与新配置写入合并为一条本地编辑历史。
- [ ] 同时存在数据层与遗留 `props.data` 时数据层为唯一生效数据源。
- [ ] 未迁移组件保存后 `props.data` 不被静默删除或改写。

## 数据解析器

- [ ] 解析器为纯函数，不发起 IO、不产生副作用。
- [ ] 解析管线依次执行数据路径提取、字段映射、逻辑层处理。
- [ ] 输出为规范化图表数据（`{ name, value }` 条目列表）。
- [ ] 嵌套数据路径可正确提取目标数组。
- [ ] 数据路径不存在返回结构化错误。
- [ ] 非数组输入返回结构化错误。
- [ ] 映射字段缺失返回结构化错误。
- [ ] 非数值映射值返回结构化错误。
- [ ] 各类错误原因可区分。
- [ ] 解析器不抛出未捕获异常。
- [ ] 错误信息可读且不泄露原始数据全文。
- [ ] 合法空数组返回空结果而非错误。
- [ ] 未配置字段映射时按 `name`→维度、`value`→数值默认推断。
- [ ] 排序作用于映射结果副本，原始数据引用不变。
- [ ] 条数限制正确截断。
- [ ] 空逻辑配置透传映射结果。
- [ ] 静态与 API 响应复用同一解析管线入口。

## 静态数据源绑定

- [ ] 可为 bar-chart 配置静态数据与字段映射。
- [ ] 图表使用数据源解析结果渲染，而非 `props.data` 直塞。
- [ ] 非法 JSON 提交被拒绝并给出可读错误。
- [ ] 非法数据结构提交被拒绝且不写入组件。
- [ ] 校验失败时画布保持上一份有效渲染。
- [ ] 合法配置提交产生一条本地编辑历史。
- [ ] 保存并重新加载后数据层配置完整保留。
- [ ] 公开预览渲染同一静态数据解析结果。
- [ ] 属性面板不再提供直接编辑 `props.data` 的 JSON textarea。

## API 数据源绑定

- [ ] 可为 bar-chart 配置 GET API 数据源（URL、查询参数、请求头、刷新间隔）。
- [ ] 非法 URL 被共享 Schema 拒绝。
- [ ] 请求测试展示响应状态码与截断响应预览。
- [ ] 请求测试失败展示可读错误（网络错误、CORS、非 2xx）。
- [ ] 请求测试不写入组件配置、不产生本地编辑历史。
- [ ] 字段映射可选值来自响应样本推断。
- [ ] 数据路径支持嵌套响应结构。
- [ ] 提交后画布图表使用真实 API 数据渲染。
- [ ] 请求由浏览器直接发起，未建设服务端代理。
- [ ] 阶段 2 未实现的方法（如 POST）返回结构化"不支持"状态且不发请求。
- [ ] 配置 UI 不暴露未实现方法的入口。

## 加载、错误与空状态

- [ ] API 请求进行中图表展示加载态。
- [ ] 加载态组件尺寸不发生变化。
- [ ] 加载中选择、拖拽、缩放组件等画布交互不受影响。
- [ ] 请求失败展示可读错误态。
- [ ] 错误态组件不从画布消失、不导致画布崩溃。
- [ ] 错误态下属性面板仍可编辑该组件配置。
- [ ] 配置修正后图表自动恢复渲染。
- [ ] 错误态不泄露原始响应全文。
- [ ] 解析成功零条展示空数据态而非错误态。
- [ ] 静态空数据与 API 空数据展示统一空态。
- [ ] 编辑器与公开预览三态表现一致。

## 定时刷新

- [ ] 刷新间隔大于零时按间隔重新请求。
- [ ] 间隔为零或未配置时不启动定时刷新。
- [ ] 配置变更时重建计时器。
- [ ] 间隔改为零时停止刷新。
- [ ] 新请求触发时中止旧请求。
- [ ] 旧响应晚到不覆盖新数据。
- [ ] 组件删除时取消计时器与进行中请求。
- [ ] 数据源类型切换时取消计时器与进行中请求。
- [ ] 页面卸载时取消计时器与进行中请求。
- [ ] 不存在未处理的 Promise rejection。
- [ ] ESLint 无新增浮动 Promise 报告。
- [ ] 公开预览中的图表同样按间隔刷新。
- [ ] 离开预览页后不再发起请求。

## 敏感请求头保护

- [ ] 敏感键名识别规则由 `@nebula/shared` 统一定义。
- [ ] 前后端均引用共享规则，无第二份实现。
- [ ] 识别覆盖 authorization、cookie、x-api-key 等键名。
- [ ] 识别对键名大小写不敏感。
- [ ] 公开预览响应中敏感请求头的值不携带明文。
- [ ] 非敏感请求头在公开预览中不受影响。
- [ ] 无数据源组件的公开预览数据不受影响。
- [ ] 受保护详情接口返回完整请求头配置。
- [ ] 公开预览不携带被脱敏请求头发起请求。
- [ ] 公开预览不使用占位值伪造请求头。
- [ ] 脱敏导致请求失败时图表进入错误态。
- [ ] 编辑器内请求不受脱敏影响。
- [ ] 脱敏不影响保存、发布和冲突恢复既有语义。

## 画布配置进入历史

- [ ] 历史条目同时记录组件数组与画布配置。
- [ ] 修改画布宽度可撤销、可重做。
- [ ] 修改画布高度可撤销、可重做。
- [ ] 修改背景颜色可撤销、可重做。
- [ ] 修改背景图片可撤销、可重做。
- [ ] 修改缩放适配模式可撤销、可重做。
- [ ] 撤销/重做同时恢复组件与画布，不出现错配。
- [ ] 无实际变化的画布提交不产生空历史记录。
- [ ] 连续型输入一次业务修改只产生一条历史。
- [ ] 历史容量限制语义不变。
- [ ] `loadProject` 清空历史的语义不变。
- [ ] 既有组件编辑历史的撤销/重做行为不回退。

## 属性面板四层组织

- [ ] 选中 bar-chart 时属性面板按数据、逻辑、视觉、交互四层分组展示。
- [ ] 非图表组件面板不出现数据源分组。
- [ ] 既有位置、样式编辑归入对应分组且行为不回退。
- [ ] 修改视觉层无需接触数据层控件。
- [ ] 修改数据层无需接触视觉层控件。
- [ ] 未配置数据源的图表提供明确配置入口与引导。
- [ ] 字段映射修改只写数据层。
- [ ] 逻辑层修改不改变数据层与视觉层配置。
- [ ] 交互层开关只写交互层。
- [ ] 悬停提示开启后编辑器与预览均可观察提示行为。
- [ ] 悬停提示默认关闭时视觉与既有行为一致。
- [ ] 每层有效提交各产生一条本地编辑历史。
- [ ] 撤销任一层的修改不影响其他层当前值。

## 单元与集成测试

- [ ] 数据层 Schema 测试覆盖合法配置与各类非法值。
- [ ] 逻辑层 Schema 测试覆盖排序、条数限制与非法值。
- [ ] 交互层与 bar-chart 视觉 props Schema 有校验测试。
- [ ] 旧组件与旧项目数据的向后兼容解析测试通过。
- [ ] 敏感键名识别测试覆盖大小写变体与非敏感键。
- [ ] 数据路径与字段映射解析测试覆盖全部错误分支。
- [ ] 逻辑层处理测试覆盖排序、截断与引用不变。
- [ ] 统一解析入口测试覆盖静态与 API 响应两条路径。
- [ ] 静态数据解析 Hook 测试覆盖配置变更与未配置状态。
- [ ] API 请求 Hook 测试覆盖成功、失败、中止与无浮动 Promise。
- [ ] 定时刷新测试覆盖假计时器触发、重建与清理。
- [ ] 竞态测试覆盖乱序响应不覆盖新数据。
- [ ] 兼容读取与一次性迁移测试覆盖迁移合并为一条历史。
- [ ] editor-store 历史条目结构测试覆盖组件与画布同步恢复。
- [ ] updateCanvas 入栈测试覆盖各字段与无变化跳过。
- [ ] 属性面板测试覆盖四层分组、校验反馈与连续输入单条历史。
- [ ] renderer 测试覆盖三态与悬停提示。
- [ ] 后端脱敏测试覆盖敏感头不含明文与受保护接口完整返回。
- [ ] 请求测试不入历史有回归测试。
- [ ] 集成测试不再全部 mock 掉数据解析与请求接线。

## Playwright E2E

- [ ] 每个数据源 E2E 独立创建项目数据。
- [ ] 每个数据源 E2E 清理自身项目数据。
- [ ] E2E 不依赖固定项目 ID、用例顺序或残留登录态。
- [ ] 静态数据源配置到渲染全链路有浏览器级覆盖。
- [ ] 静态数据源保存、重载与公开预览一致有浏览器级覆盖。
- [ ] API 配置、请求测试、响应预览与字段映射有浏览器级覆盖。
- [ ] API 失败错误态有浏览器级覆盖。
- [ ] API 空数据态有浏览器级覆盖。
- [ ] 定时刷新发生与离开页面停止有浏览器级覆盖。
- [ ] 修改视觉层后数据层配置不变有浏览器级覆盖。
- [ ] 修改数据层后视觉层配置不变有浏览器级覆盖。
- [ ] 各层修改可独立撤销有浏览器级覆盖。
- [ ] 画布尺寸、背景、适配方式修改的撤销与重做有浏览器级覆盖。
- [ ] 公开预览数据不含敏感请求头明文有端到端证据。
- [ ] API Mock 不依赖真实外部网络。
- [ ] 失败时保留 trace、截图或等价定位证据。
