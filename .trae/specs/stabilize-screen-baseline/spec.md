# 大屏可靠性与事实基线 Spec

## Why

现有大屏设计器已经具备项目创建、编辑、保存、发布和公开预览链路，但审计确认关键边界仍不稳定：公开预览接口可读取草稿；独立编辑器路由未纳入前端认证路由树；发布动作只切换状态，未明确保存内容与发布内容的边界；编辑器与预览重复维护组件容器样式，已出现旋转缺失；属性输入的草稿状态可能在画布变换后继续显示旧值；保存请求未携带保存基线，多个客户端会发生最后写入覆盖；前端也没有可恢复的冲突界面。与此同时，当前测试主要覆盖局部纯函数和后端基础 CRUD，尚未形成覆盖这些风险的可信基线。

本 Spec 是 `evolve-screen-design-platform` 阶段 0「可靠性与事实基线」的实施规格。目标是在不引入项目成员权限、服务端修订版或实时协作的前提下，先稳定当前编辑、保存、发布和预览链路，为后续演进提供可验证的安全与一致性基础。

## What Changes

- 将“受保护项目接口”和“公开预览接口”定义为两个互不混用的数据边界。
- 公开预览只允许读取已发布项目，草稿、已删除或不存在项目均不得返回项目内容。
- 将大屏列表和编辑器路由纳入认证路由边界，公开预览路由保持匿名可访问。
- 明确保存与发布的职责：保存写入草稿内容并将项目状态设为 `draft`；已发布项目保存后立即退出公开可见状态，必须再次发布才可公开预览；发布只发布服务端已经成功保存的当前内容，不隐式接收编辑器未保存状态。
- 提取编辑器与预览共用的组件容器样式解析能力，统一位置、尺寸、层级、旋转、透明度、边框、背景、圆角、溢出和可见性语义。
- 明确属性输入草稿与编辑器 Store 的同步规则，避免画布拖拽、缩放、旋转或切换选中对象后继续展示或提交旧值。
- 将服务端响应的 `updatedAt` 作为阶段 0 的保存基线值，更新和发布请求统一通过 `expectedUpdatedAt` 提交客户端最后确认的基线。
- 服务端以原子条件写入比较数据库 `updatedAt` 与请求 `expectedUpdatedAt`，基线过期时返回专用冲突业务错误，不允许静默覆盖。
- 前端提供阻塞式冲突 UI，保留本地未保存内容，并提供重新加载服务端版本和取消关闭两种明确选择。
- 建立后端单元测试、前端组件/Hook 测试和关键 E2E 测试基线，并重新记录实际执行结果。
- 本 Spec 只定义规格、任务和验收标准，不实施产品代码。

## ADDED Requirements

### Requirement: 公开预览与受保护读取隔离

系统 SHALL 将公开预览建模为独立读取能力，而不是复用不校验发布状态的受保护项目详情查询。

#### Scenario: 已发布项目可匿名预览

- **GIVEN** 项目状态为 `published`
- **WHEN** 未认证访问者请求公开预览接口
- **THEN** 服务端返回该项目的可预览数据
- **AND** 请求不要求访问令牌
- **AND** 返回结果通过共享项目 Schema 校验

#### Scenario: 草稿项目不可匿名预览

- **GIVEN** 项目状态为 `draft`
- **WHEN** 任意访问者请求公开预览接口
- **THEN** 服务端不得返回项目画布、组件、名称或其他项目内容
- **AND** 返回不泄露草稿内容的未找到类错误
- **AND** 前端公开预览页显示统一的“不存在或未发布”空状态

#### Scenario: 公开预览不扩大权限

- **GIVEN** 一个项目已经发布
- **WHEN** 匿名访问者获得公开预览数据
- **THEN** 该访问者仍不能调用项目列表、受保护详情、创建、更新、发布或删除接口
- **AND** 公开预览数据不得被当作编辑器初始化数据源

### Requirement: 认证路由边界

系统 SHALL 在前端路由层明确区分受保护的大屏管理/编辑路由和公开预览路由，并以服务端认证作为最终安全边界。

#### Scenario: 未认证用户访问大屏列表

- **WHEN** 未认证用户访问大屏列表路由
- **THEN** 前端在加载项目数据前重定向到登录页
- **AND** 登录后可返回原目标或进入已定义的受保护入口

#### Scenario: 未认证用户直接访问编辑器

- **WHEN** 未认证用户直接访问某个大屏编辑器 URL
- **THEN** 前端在挂载编辑器和请求受保护详情前重定向到登录页
- **AND** 不闪现项目名称、画布或组件内容

#### Scenario: 公开预览保持匿名入口

- **WHEN** 未认证用户访问公开预览路由
- **THEN** 前端不得因全局认证守卫将其重定向到登录页
- **AND** 页面只调用公开预览接口

#### Scenario: 服务端拒绝未认证受保护请求

- **WHEN** 未认证客户端绕过前端直接调用项目列表、详情、创建、更新、发布或删除接口
- **THEN** 服务端返回 401
- **AND** 本阶段不提前实现所有者、编辑者、查看者项目成员矩阵

### Requirement: 保存与发布边界

系统 SHALL 将“保存当前编辑内容”和“发布服务端已保存内容”定义为两个显式操作，避免发布动作隐式接收或覆盖未保存编辑状态。

#### Scenario: 保存更新可编辑内容并回到草稿

- **GIVEN** 用户在编辑器中修改了画布或组件
- **WHEN** 用户执行保存且保存基线有效
- **THEN** 服务端更新项目可编辑字段
- **AND** 响应返回新的 `updatedAt`
- **AND** 服务端将项目状态设为 `draft`
- **AND** 若项目保存前为 `published`，保存成功后公开预览立即不可用，直至用户基于新保存内容再次发布

#### Scenario: 发布只作用于已保存版本

- **GIVEN** 编辑器存在尚未成功保存的本地修改
- **WHEN** 用户尝试发布
- **THEN** 系统不得把这些本地修改作为发布请求的隐式请求体
- **AND** 前端必须要求先完成保存，或明确阻止发布并提示存在未保存修改
- **AND** 只有保存成功后获得的新基线可用于发布

#### Scenario: 发布成功建立公开可见边界

- **GIVEN** 项目当前为草稿且客户端发布基线与服务端一致
- **WHEN** 用户执行发布
- **THEN** 服务端将项目状态更新为 `published`
- **AND** 返回新的 `updatedAt`
- **AND** 后续公开预览读取到此次已保存并已发布的服务端内容

#### Scenario: 过期基线不得发布

- **GIVEN** 客户端持有的 `updatedAt` 已落后于服务端
- **WHEN** 客户端执行发布
- **THEN** 服务端返回保存冲突错误
- **AND** 不改变项目发布状态
- **AND** 不将旧客户端内容或状态覆盖到服务端

### Requirement: 编辑器与预览共享组件容器样式

系统 SHALL 通过单一、可测试的共享样式解析能力生成编辑器和预览的组件容器样式，禁止两个场景分别手写同一组公共样式规则。

#### Scenario: 公共样式字段一致

- **GIVEN** 组件包含位置、尺寸、层级、旋转、透明度、边框、背景、圆角和溢出配置
- **WHEN** 该组件分别在编辑器和公开预览中渲染
- **THEN** 两处使用同一解析结果表达这些字段
- **AND** 相同项目数据产生相同的容器几何与视觉样式

#### Scenario: 旋转在预览中生效

- **GIVEN** 组件的 `position.rotation` 为非零值
- **WHEN** 用户保存、发布并打开公开预览
- **THEN** 预览组件使用与编辑器相同的旋转角度和变换语义
- **AND** 旋转不被画布缩放覆盖

#### Scenario: 隐藏组件不参与预览

- **GIVEN** 组件状态为隐藏
- **WHEN** 公开预览渲染项目
- **THEN** 该组件不出现在可见输出中
- **AND** 共享样式解析不把编辑器专用选中框、辅助线或交互状态带入预览

#### Scenario: 编辑器专用样式保持隔离

- **GIVEN** 编辑器需要选中态、边框辅助线或 Moveable 控件
- **WHEN** 编辑器渲染组件容器
- **THEN** 公共样式先由共享解析能力生成
- **AND** 编辑器专用样式在公共样式之外叠加
- **AND** 公开预览不得依赖编辑器 Store 或交互组件

### Requirement: 属性面板与画布状态同步

系统 SHALL 以编辑器 Store 中当前组件数据作为属性面板的权威值，并为输入草稿定义可预测的外部更新处理规则。

#### Scenario: 非编辑状态实时跟随画布变换

- **GIVEN** 用户未聚焦某个属性输入框
- **WHEN** 同一组件通过拖拽、缩放或旋转更新位置与尺寸
- **THEN** 属性面板在下一次 React 提交中显示 Store 的最新值
- **AND** 不继续显示组件变换前的旧值

#### Scenario: 编辑中的草稿不被无关渲染重置

- **GIVEN** 用户正在某个属性输入框中输入尚未提交的草稿
- **WHEN** 页面发生与该属性无关的重渲染
- **THEN** 当前草稿保持不变
- **AND** 不产生额外历史记录或 Store 写入

#### Scenario: 外部值变化使旧草稿失效

- **GIVEN** 用户正在编辑组件 A 的属性草稿
- **WHEN** 用户切换到组件 B，或组件 A 的同一属性被画布操作更新
- **THEN** 输入控件不得在新上下文中提交旧草稿
- **AND** 控件按明确策略放弃旧草稿并显示新的权威值
- **AND** Escape、Enter、Blur 和方向键提交行为保持确定且有测试覆盖

#### Scenario: 属性提交只产生一次业务更新

- **GIVEN** 用户直接输入一个新数值
- **WHEN** 用户通过 Enter 或 Blur 提交
- **THEN** 只触发一次有效的属性更新
- **AND** 无效、空白或未变化输入不写入 Store
- **AND** 不因 Enter 后触发 Blur 而重复提交

### Requirement: 基于 expectedUpdatedAt 的保存基线契约

系统 SHALL 使用请求字段 `expectedUpdatedAt` 提交客户端最后确认的服务端 `updatedAt`，并对更新与发布执行服务端原子比较。

#### Scenario: 更新请求必须携带保存基线

- **GIVEN** 客户端已加载一个项目
- **WHEN** 客户端提交更新
- **THEN** 请求包含 `expectedUpdatedAt`，其值等于客户端最后确认的服务端 `updatedAt`
- **AND** `expectedUpdatedAt` 只用于并发条件，不作为客户端可任意写入的目标时间
- **AND** 服务端响应返回写入后的新 `updatedAt`

#### Scenario: 基线匹配时原子保存成功

- **GIVEN** 请求 `expectedUpdatedAt` 与数据库当前 `updatedAt` 一致
- **WHEN** 服务端执行更新
- **THEN** 服务端通过项目 ID 且数据库 `updatedAt` 等于请求 `expectedUpdatedAt` 的单次条件写入完成保存
- **AND** 受影响记录数为一
- **AND** 客户端以响应中的 `updatedAt` 替换旧保存基线

#### Scenario: 基线过期时拒绝覆盖

- **GIVEN** 另一个客户端已经更新项目并产生新的 `updatedAt`
- **WHEN** 客户端使用旧值作为 `expectedUpdatedAt` 提交更新
- **THEN** 条件写入不修改任何记录
- **AND** 服务端返回专用的保存冲突业务码和 HTTP 409
- **AND** 服务端不自动合并、不执行最后写入覆盖

#### Scenario: 冲突与资源不存在可区分

- **WHEN** 条件更新未命中记录
- **THEN** 服务端能够区分项目不存在与项目基线过期
- **AND** 不存在返回项目未找到错误
- **AND** 基线过期返回保存冲突错误

#### Scenario: 时间格式保持契约一致

- **WHEN** `expectedUpdatedAt` 在共享请求 Schema 和 HTTP 请求中传递，且 `updatedAt` 在服务端比较和响应中传递
- **THEN** 两者使用现有日期时间字符串契约
- **AND** 不依赖客户端本地时区格式化结果进行比较

### Requirement: 保存冲突 UI

系统 SHALL 为保存和发布冲突提供可恢复、不可静默忽略的界面，并在用户作出选择前保留本地编辑内容。

#### Scenario: 保存冲突打开阻塞式界面

- **GIVEN** 保存请求返回专用冲突业务码
- **WHEN** 前端处理该错误
- **THEN** 不只显示瞬时 Toast
- **AND** 打开具有明确标题、原因和影响说明的阻塞式冲突界面
- **AND** 当前编辑器 Store 中的本地修改保持不变

#### Scenario: 用户取消冲突处理

- **GIVEN** 冲突界面已打开
- **WHEN** 用户选择取消或继续留在当前编辑器
- **THEN** 冲突界面关闭或保持可再次操作
- **AND** 本地内容不丢失
- **AND** 保存基线不被伪装成最新值

#### Scenario: 用户重新加载服务端版本

- **GIVEN** 冲突界面已打开
- **WHEN** 用户确认重新加载服务端版本
- **THEN** 前端重新请求受保护项目详情
- **AND** 成功响应整体替换编辑器项目和保存基线
- **AND** 选中态、临时输入草稿和未提交本地历史按定义重置
- **AND** 界面明确提示本地未保存修改将被放弃

#### Scenario: 发布冲突复用同一处理模型

- **GIVEN** 发布请求因基线过期而冲突
- **WHEN** 前端处理错误
- **THEN** 使用与保存冲突一致的冲突识别和恢复入口
- **AND** 不错误显示“发布成功”
- **AND** 公开预览仍保持服务端冲突前的发布状态和内容

### Requirement: 测试事实基线

系统 SHALL 使用实际可执行的自动化测试和命令结果建立阶段 0 基线，不以旧文档中的历史数字作为完成证据。

#### Scenario: 后端边界测试

- **WHEN** 阶段 0 完成
- **THEN** 后端测试覆盖已发布项目公开预览成功、草稿公开预览失败、未认证受保护接口失败、基线匹配更新成功、基线过期冲突、项目不存在和过期发布冲突
- **AND** 测试验证未命中条件写入时数据库内容未改变

#### Scenario: 前端单元与组件测试

- **WHEN** 阶段 0 完成
- **THEN** 前端测试覆盖共享样式解析、旋转样式、属性输入外部同步、单次提交、保存基线更新和冲突错误分流
- **AND** 冲突 UI 测试验证取消保留本地内容、重新加载替换本地内容

#### Scenario: 浏览器关键路径测试

- **WHEN** 阶段 0 完成
- **THEN** E2E 覆盖未认证编辑器重定向、匿名公开预览、草稿不可预览、保存后发布、已发布内容可预览以及双客户端旧基线保存冲突
- **AND** E2E 不依赖测试之间的执行顺序

#### Scenario: 重新记录当前事实

- **WHEN** 进入阶段 0 验收
- **THEN** 执行并记录适用的 typecheck、lint、Biome、后端测试、前端测试和 E2E 命令结果
- **AND** 记录实际测试文件数、用例数、通过数、失败数和跳过数
- **AND** 若命令未执行或失败，不得将相应检查项标记完成

## MODIFIED Requirements

### Requirement: 项目更新接口

现有项目更新接口 SHALL 从“按 ID 直接覆盖”修改为“按 ID 且数据库 `updatedAt` 等于请求 `expectedUpdatedAt` 执行条件更新，并将状态设为 `draft`”。

#### Scenario: 兼容现有可编辑字段

- **WHEN** 客户端提交名称、描述、画布、组件或缩略图更新
- **THEN** 现有字段语义保持不变
- **AND** 新增 `expectedUpdatedAt` 作为必需并发条件
- **AND** 保存已发布项目后状态降为 `draft`，公开预览在再次发布前不可读取新保存内容
- **AND** 成功响应继续返回完整项目数据

### Requirement: 项目发布接口

现有发布接口 SHALL 从“按 ID 直接设置状态”修改为“在请求 `expectedUpdatedAt` 与服务端 `updatedAt` 匹配时发布服务端当前草稿内容”。

#### Scenario: 发布不承担保存职责

- **WHEN** 客户端调用发布接口
- **THEN** 请求只表达项目 ID 与 `expectedUpdatedAt`
- **AND** 不接受画布或组件内容作为隐式保存负载
- **AND** 发布成功后返回完整项目及新 `updatedAt`

### Requirement: 公开预览接口

现有公开预览接口 SHALL 从复用普通详情查询修改为使用只返回已发布项目的专用服务能力。

#### Scenario: 状态条件不可旁路

- **WHEN** 公开预览查询项目
- **THEN** 数据库查询条件包含项目 ID 与 `published` 状态
- **AND** 控制器不得改回调用不校验状态的普通详情方法

## REMOVED Requirements

### Requirement: 匿名访问草稿项目

系统 SHALL NOT 允许公开预览接口返回草稿项目。

### Requirement: 最后写入覆盖

系统 SHALL NOT 在客户端保存基线过期时继续更新或发布项目。

### Requirement: 编辑器与预览分别维护公共样式

系统 SHALL NOT 在编辑器和预览中继续复制维护组件位置、尺寸、旋转和公共视觉样式映射。

### Requirement: 冲突仅用瞬时提示处理

系统 SHALL NOT 将可能导致本地内容丢失的保存冲突仅作为自动消失的 Toast 处理。

## Out of Scope

- 项目所有者、编辑者、查看者成员权限矩阵及资源级授权。
- 服务端不可变修订版（revision）、历史比较、恢复或审计日志。
- 自动合并、字段级合并、CRDT、OT 或实时多人协作。
- 发布版本快照、草稿与线上版本长期并存的数据模型；本阶段保存会将项目统一设为 `draft`，再次发布后公开预览才读取当前已保存内容。
- 新组件、新工具、数据源运行时、工作区角色或大屏交互状态机重构。
- 对现有编辑器所有历史缺陷进行无关重构。

## Dependencies and Ordering

1. 先稳定共享错误码与保存基线 Schema，再接入后端和前端。
2. 先实现后端公开预览隔离，再开放匿名预览 E2E。
3. 先实现共享样式解析并建立测试，再替换编辑器和预览调用。
4. 先明确属性输入同步契约，再修改属性面板接入。
5. 先实现服务端原子乐观锁，再传递前端基线并接入冲突 UI。
6. 发布边界依赖保存基线和本地脏状态可判断。
7. 所有功能链完成后再重跑并记录测试事实基线。
