# Tasks

本文件是整体阶段规划，不直接展开全部代码级任务。每进入一个阶段，先为该阶段创建独立 Spec，再拆成可单独验证和回滚的原子任务。

## 小步快跑规则

- 每次只启动一个主阶段。
- 每个任务只交付一个可观察结果。
- 数据模型、共享 Schema、后端接口、前端 UI、迁移和测试应分别拆分、独立交付。
- 新能力先建立最小闭环，再扩展高级能力。
- 每个阶段必须完成自动化验证和手动验收后才能进入下一阶段。
- 不提前引入后续阶段依赖；阶段 0 先建立身份认证与公开预览边界，阶段 4 再建立项目成员授权，实时协作不得先于服务端修订版和权限隔离。
- 统一术语：“数据、逻辑、视觉、交互”四层配置、保存基线、服务端修订版（revision）、本地编辑历史、本地项目快照。

- [ ] **阶段 3：服务端修订版与恢复**
  - 目标：建立服务端修订版（revision）、修订版差异和安全恢复。
  - 用户价值：修改可追溯、可比较、可恢复，不依赖单个浏览器的本地项目快照。
  - 范围：修订版数据模型、保存生成修订版、修订版列表、结构化 diff、恢复为新修订版、本地项目快照兼容。
  - 明确不做：复杂分支、三方合并、实时光标、项目成员角色矩阵。
  - 进入条件：阶段 0 的身份认证基线完成，且阶段 2 的四层配置结构稳定。
  - 退出条件：服务端保存、比较、恢复闭环通过，操作者身份可审计，历史不可被恢复操作破坏。
  - 下一步：为阶段 3 创建独立 Spec，先修订版存储，再只读历史，再 diff，最后恢复。
- [ ] **阶段 4：权限隔离与角色工作区**
  - 目标：在阶段 0 身份认证基线上建立真正可执行的全局权限、项目成员权限和开发者/业务员工作区。
  - 用户价值：所有者、编辑者和查看者只能执行获授权操作，工作区类型不会扩大权限，项目数据不会跨用户泄露。
  - 范围：后端权限 Guard、前端权限上下文、项目所有者/编辑者/查看者、按钮/菜单/路由权限、开发者/业务员工作区预设和持久化。
  - 明确不做：实时协作同步、评论、分支合并。
  - 进入条件：阶段 3 完成，服务端修订版审计已能记录操作者。
  - 退出条件：未认证请求返回 401、权限不足返回 403、非成员资源探测返回不泄露存在性的 404，前端视图裁剪、项目成员权限和两类工作区均有 E2E。
  - 下一步：为阶段 4 创建独立 Spec，先定义权限矩阵，再后端授权，再项目成员，再前端视图，最后工作区布局。
- [ ] **阶段 5：实时协作基础**
  - 目标：实现项目房间、Presence、增量同步和评论批注。
  - 用户价值：多个有权限用户可同时编辑并感知彼此操作，断线后可恢复。
  - 范围：WebSocket 身份认证与项目成员授权、项目房间、在线成员、远端光标/选区、增量操作、修订版序号补偿、评论线程。
  - 明确不做：复杂三方分支合并、AI 自动冲突决策。
  - 进入条件：阶段 3 和阶段 4 均完成。
  - 退出条件：双浏览器协作 E2E 通过，断线重连、未认证连接和无项目权限连接均有验证。
  - 下一步：为阶段 5 创建独立 Spec，先 Presence，再只读同步，再增量写入，最后评论。
- [ ] **阶段 6：分支合并与智能辅助评估**
  - 目标：基于真实使用数据决定是否实施项目分支、三方合并和 AI 建议能力。
  - 用户价值：支持更复杂的并行创作，但不以牺牲可解释性和可靠性为代价。
  - 范围：需求验证、冲突分类、三方 diff 原型、AI 建议安全边界和审计要求。
  - 明确不做：在没有真实需求与稳定基础设施时直接上线复杂分支或自动覆盖。
  - 进入条件：阶段 5 稳定运行，并积累真实并行编辑与冲突数据。
  - 退出条件：形成实施或不实施的证据化决策；若实施，再创建独立功能 Spec。

## Task Dependencies

- 阶段 0 → 阶段 1：先保证现有行为可靠，再迁移交互状态和增加工具。
- 阶段 1 → 阶段 2：工具和画布状态稳定后再引入动态数据状态。
- 阶段 0 + 阶段 2 → 阶段 3：身份认证基线和四层配置结构稳定后，才能生成可归属操作者的可靠修订版差异。
- 阶段 3 → 阶段 4：服务端修订版审计提供权限操作的追踪基础；阶段 4 不反向阻塞阶段 3 的基础操作者记录。
- 阶段 3 + 阶段 4 → 阶段 5：实时协作依赖服务端修订版顺序和项目成员授权。
- 阶段 5 → 阶段 6：只有真实协作数据才能判断分支与 AI 辅助是否值得实施。

## 可并行边界

- 同一阶段内，互不修改同一契约的研究、测试设计和 UI 原型可并行。
- 共享 Schema 与依赖它的前后端实现不可并行落地，必须先稳定契约。
- 后端权限执行与前端视图裁剪可在共享权限契约确定后并行。
- Presence 与评论数据模型可在阶段 5 内并行研究，但接入同一 WebSocket 协议时需串行整合。
