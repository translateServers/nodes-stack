# Checklist

本 checklist 用于验收 `event-blueprint` 实现。所有实施条目默认未完成；只有产品行为、自动化验证和必要浏览器证据均存在时才能勾选。

## 规格边界

- [x] 实施范围只覆盖蓝图契约、编译器、运行时、可视化编辑器、历史与保存集成、模拟调试、双向联动、模板与高级触发/动作。
- [x] 未引入自定义 JS 脚本节点或任意代码执行。
- [x] 未引入数据流引脚连线。
- [x] 未引入 WebSocket 等实时触发源或多客户端事件广播。
- [x] 未引入服务端修订版（revision）、项目分支、实时多人协作或权限差异。
- [x] 未引入 AI 生成规则或智能推荐。
- [x] 未引入云端模板库或蓝图文件导入导出。
- [x] `interval` 触发器仅预览会话内有效，未做服务端调度。
- [x] 除 `@xyflow/react` 外未引入其它新依赖。
- [x] 未一次性重写编辑器 Store、属性面板或预览链路。

## 阶段进入门禁

- [x] 阶段 2 checklist 已全部完成。
- [x] 阶段 2 验证记录存在且可追溯。
- [x] 阶段 2 关键回归当前有效。
- [x] 任一阶段 2 门禁失效时已停止并先恢复基线。

## 实施基线

- [x] 已记录占位骨架、无 `blueprint` 字段、两要素历史快照、无预览事件运行时的实施前事实。
- [x] 已执行实施前前端 screen 定向测试并记录实际结果。
- [x] 已执行实施前后端 screen 定向测试并记录实际结果。
- [x] 已执行实施前共享包测试与 screen Playwright E2E 并记录实际结果。
- [x] 已执行实施前 typecheck、lint 和 Biome 并记录实际结果。
- [x] 蓝图编辑 UI 使用稳定、可访问的定位契约。
- [x] API 相关 E2E 使用 route Mock，不依赖真实外部网络。

## 蓝图契约

- [x] `blueprint` 为 `ScreenProjectSchema` 可选字段，含 `version: 1`、`nodes[]`、`edges[]`。
- [x] 节点按 trigger/condition/action/comment 判别联合校验。
- [x] 边仅承载执行流，含 source/target 及引脚标识。
- [x] 未知节点 kind 或动作类型被拒绝并给出可读错误。
- [x] `navigate.url` 仅允许 http/https 协议。
- [x] 无 `blueprint` 的旧项目解析成功且行为不回退。
- [x] 未编辑蓝图的项目保存时不凭空写入 `blueprint` 字段。
- [x] 服务端保存使用同一共享 Schema 校验，非法蓝图不写入数据库。
- [x] `expectedUpdatedAt` 保存基线语义不变。

## 编译器

- [x] 编译器为纯函数，不发起 IO、不产生副作用。
- [x] 以 trigger 为入口按拓扑序展开 `CompiledRule[]`。
- [x] comment 节点与未连接子图被排除并产出 info 诊断。
- [x] 执行流环返回 error 诊断并定位环。
- [x] 含环触发器不产出规则，其余触发器不受影响。
- [x] 悬空 componentId 返回 warning 级 dangling 诊断。
- [x] dangling 节点与规则不被静默删除。
- [x] 空参数返回 error 诊断并定位到节点与字段。
- [x] 诊断分级（error/warning/info）且消息可读。

## 运行时

- [x] 编辑器画布点击/悬停不触发蓝图动作。
- [x] 公开预览中组件点击与页面加载按编译结果触发。
- [x] 动作按拓扑序执行。
- [x] 链式触发深度超过 10 层即截断并告警，不死循环。
- [x] 单条规则内失败动作不中断后续独立动作。
- [x] `setVisibility` 作用于预览可见性覆盖表，不改写项目数据。
- [x] `refreshDataSource` 复用取消协议与竞态防护，无浮动 Promise。
- [x] `navigate` 仅按白名单协议打开。
- [x] `scrollToComponent` 平滑滚动至目标组件。
- [x] dangling 动作运行时跳过并记录。
- [x] 页面卸载时中止进行中动作、清理计时器与请求。
- [x] 不存在未处理的 Promise rejection。

## 可视化编辑器

- [ ] 占位骨架与"功能开发中"文案已被真实编辑器替换。
- [ ] 入口（项目菜单·工具 → 事件蓝图）与底部 Sheet 容器规格不变。
- [x] 拖出连线时兼容引脚高亮磁吸、不兼容引脚置灰。
- [x] 连线松手落空白弹出搜索面板，插入后自动完成连线。
- [x] 双击空白呼出搜索面板，支持键盘选择与 Enter 插入。
- [x] 支持点选、Shift 多选、框选、Ctrl+A 与多选整体拖拽。
- [x] 节点移动吸附 8px 网格并提供节点间对齐吸附线。
- [x] 滚轮以光标为中心缩放（0.25x–2x），Space+拖拽平移。
- [x] Fit View 与缩放到选区可用。
- [x] 节点按分类着色并显示组件名称与类型图标。
- [ ] Ctrl+C/X/V 与 Ctrl+D 可用，粘贴重新生成节点 id。
- [ ] 剪贴板内容经共享 Schema 校验，支持跨项目粘贴。
- [ ] 非法剪贴板内容给出可读提示，不产生脏节点。

## 历史与保存

- [x] 历史条目同时记录组件数组、画布配置与 `blueprint`。
- [x] 撤销/重做同步恢复三者，不出现蓝图与组件错配。
- [x] 无 `blueprint` 的旧快照按空蓝图处理。
- [ ] 节点增删、连线增删、参数修改、布局拖拽结束各产生一条历史。
- [ ] 拖拽中间态不入栈，一次拖拽只产生一条历史。
- [ ] 无实际变化的提交不产生空历史记录。
- [x] 历史容量限制与 `loadProject` 清空历史的语义不变。
- [ ] Sheet 内撤销/重做与编辑器全局同一本地编辑历史。
- [ ] `blueprint` 随项目保存与发布。
- [ ] 存在 error 级诊断时发布给出确认提示与摘要。
- [ ] 草稿项目的蓝图不通过公开预览暴露。

## 校验与问题面板

- [ ] 图编辑后诊断实时刷新且经 rAF 节流。
- [ ] 问题节点在画布上标记，修复后标记消失。
- [ ] 问题面板按 error/warning/info 分级列出。
- [ ] 点击诊断条目定位并闪烁聚焦对应节点。
- [ ] error 级诊断不阻塞保存。
- [ ] error 级诊断对应的触发器在预览运行时不执行。

## 模拟调试（M2）

- [ ] Sheet 内可对选中 trigger 节点执行模拟触发。
- [ ] 沙盒执行不污染预览与画布真实状态。
- [ ] 执行路径边流动高亮、节点依次亮起，结束后复位。
- [ ] 日志按序展示动作执行/跳过结果、原因与耗时。
- [ ] 失败动作红色标记并可点击定位节点。

## 双向联动与模板（M2）

- [ ] 点击蓝图节点时主画布闪烁高亮对应组件。
- [ ] 画布选中组件时蓝图过滤展示涉及该组件的节点与链路。
- [ ] 组件重命名时节点标签实时跟随。
- [ ] 空蓝图提供一键模板，插入经 Schema 校验并产生一条历史。
- [ ] 小地图与多选对齐分布工具可用。

## 高级触发与动作（M3）

- [ ] condition 节点 then/else 双输出引脚按表达式分流执行。
- [ ] 表达式构建器不产生自定义脚本输入入口。
- [ ] `dataLoaded`/`dataError`/`componentHover`/`interval` 触发器按语义触发。
- [ ] `interval` 仅预览会话内有效，页面卸载即清理。
- [ ] `requestApi` 沿用阶段 2 取消协议与竞态防护。
- [ ] 公开预览中敏感请求头按共享规则脱敏，失败进入失败态不伪造。
- [ ] 动作参数模板插值只读求值，无代码执行路径。

## 安全与公开预览

- [ ] 公开预览按编译结果执行蓝图。
- [ ] 公开预览数据范围不因蓝图引入新的明文敏感信息。
- [ ] 编辑器模拟沙盒与公开预览的动作语义、深度截断与降级行为一致。
- [ ] 阶段 2 脱敏链路回归不回退。

## 性能与工程规范

- [ ] 200+ 节点蓝图仅渲染可视区域，交互帧率不低于 50fps。
- [ ] 节点组件 memo 化，边重算经 rAF 节流。
- [ ] Sheet 打开期间画布全局快捷键挂起，关闭后恢复。
- [ ] Esc 分层顺序为取消连线 → 取消选择 → 关闭 Sheet。
- [x] 代码通过 TypeScript strict 与 ESLint `recommendedTypeChecked`。
- [x] 代码符合 Biome 格式规范。
- [ ] 编辑器外壳组件使用 shadcn/ui。

## 单元与集成测试

- [x] 蓝图 Schema 测试覆盖合法配置与各类非法值。
- [x] URL 协议白名单测试覆盖 `javascript:` 等非法协议。
- [x] 向后兼容测试覆盖无 `blueprint` 旧项目解析。
- [x] 编译器测试覆盖拓扑、环、 dangling、空参数与诊断分级。
- [x] 引擎测试覆盖规则匹配、深度截断边界与动作计划。
- [x] 执行器测试覆盖四种动作语义与 dangling 跳过。
- [x] refreshDataSource 测试覆盖取消、乱序响应与卸载清理。
- [x] editor-store 测试覆盖三要素历史同步恢复与旧快照兼容。
- [ ] 蓝图编辑 action 测试覆盖单条历史与空提交跳过。
- [ ] 剪贴板测试覆盖序列化、重新生成 id 与非法内容拒绝。
- [x] 搜索面板与引脚兼容判定有组件测试。
- [ ] 沙盒调试测试覆盖不污染真实状态。
- [x] 集成测试不再全部 mock 掉编译与执行接线。

## Playwright E2E

- [ ] 每个蓝图 E2E 独立创建项目数据。
- [ ] 每个蓝图 E2E 清理自身项目数据。
- [ ] E2E 不依赖固定项目 ID、用例顺序或残留登录态。
- [ ] 可视化搭建到预览执行全链路有浏览器级覆盖。
- [ ] 四种动作与 pageLoad 触发有浏览器级覆盖。
- [ ] 深度截断与 dangling 有浏览器级覆盖。
- [ ] 模拟调试链路有浏览器级覆盖。
- [ ] 双向联动与模板插入有浏览器级覆盖。
- [ ] condition 分支与高级触发器有浏览器级覆盖。
- [ ] requestApi 使用 route Mock，不依赖真实外部网络。
- [ ] 失败时保留 trace、截图或等价定位证据。
