# 单用户交互闭环 Spec

## Why

阶段 0 已稳定大屏项目的认证、保存、发布、预览、样式一致性和并发冲突边界，阶段 1 可以开始处理编辑器内部的单用户交互一致性。

当前编辑器已经具备 Moveable、Selecto、画布平移、锚点缩放、Alt 拖拽复制、分组进入、图层排序和快捷键等基础能力，也已经存在交互状态机、工具状态机、修饰键 Hook 和事件路由纯函数。但是审计确认这些模块尚未形成生产闭环：`useInteractionStateMachine` 仍是未接入画布的 API 骨架；`activeTool` 只被状态栏和快捷键消费，没有控制画布；选择、拖拽、缩放、旋转和平移继续由多个局部布尔值、ref 和事件回调独立决定；文字、矩形、椭圆、图片、吸管和缩放工具尚未全部产生可观察结果；文本编辑状态在工具状态机和交互状态机中存在重复设计；现有 Playwright 测试也没有覆盖关键画布组合交互。

本 Spec 是 `evolve-screen-design-platform` 阶段 1「单用户交互闭环」的独立实施规格。阶段 0 的 checklist、验证记录和关键回归全部有效是本阶段的强制进入条件；任一阶段 0 门禁失效时，阶段 1 不得开始或继续。实施采用“小步快跑”和“状态定义 → 单场景接入 → 删除对应旧状态”的渐进策略，保留现有 Moveable、Selecto、事件路由和高频反馈 Store，不进行一次性画布重写。

## What Changes

- 建立编辑器会话级交互控制器，统一组合当前工具、临时工具栈、交互状态、文本编辑对象和工具能力，不将 DOM ref 或帧级手势数据塞入持久项目 Store。
- 建立单一工具注册表，统一工具 ID、名称、图标、`shortcutId`、光标、能力和实现状态，供工具入口、状态栏、快捷键帮助和画布共同消费；既有 `SHORTCUTS_REGISTRY` 继续唯一拥有实际键位、scope、`preventDefault` 和浏览器冲突信息。
- 扩展并生产接入交互状态机，以镜像模式验证真实 Moveable、Selecto、平移、缩放、文本编辑和右键菜单事件，再逐场景切换为交互仲裁源。
- 为所有瞬时交互建立统一结束和取消路径，覆盖正常结束、Escape、`pointercancel`、`lostpointercapture`、窗口失焦和组件卸载。
- 让选择工具和抓手工具真实控制 Selecto、Moveable 和画布平移；Space 临时抓手与主工具使用同一工具栈和画布条件。
- 完成文字工具闭环：创建文本、进入编辑、提交、取消、退出以及编辑期间的快捷键隔离，并删除重复的文本编辑状态源。
- 完成矩形和椭圆工具闭环：拖拽创建、最小尺寸、取消、默认样式、创建后选中及历史记录。
- 完成图片工具闭环：从用户选择的本地图片创建图片组件，处理取消、失败和资源生命周期，不把本地绝对路径写入项目。
- 完成缩放工具闭环：以点击位置为锚点放大，使用工具约定的反向操作缩小，并遵守缩放上下限。
- 完成吸管工具最小闭环：只从编辑器拥有且允许采样的可见表面解析颜色，更新会话活动颜色；对当前选中且支持颜色属性的组件执行一次可撤销应用，不采样跨域像素或浏览器外内容。
- 补充可访问的工具入口，使工具按钮、状态栏、快捷键帮助、光标和真实画布行为保持一致；未实现或不可用能力不得显示为已完成状态。
- 增加浏览器级编辑器交互 E2E，覆盖框选后拖拽、双击进入分组、Alt 拖拽复制、图层拖拽排序、快捷键缩放及浏览器默认行为隔离。
- 本 Spec 只定义规格、任务和验收标准；获得批准前不修改产品代码。

## Impact

- Affected specs:
  - `evolve-screen-design-platform`：细化并实施阶段 1，完成后作为阶段 2 的进入条件。
  - `research-interaction-architecture`：将既有事件路由、交互状态机、工具状态机、修饰键栈和反馈层研究接入生产路径。
  - `stabilize-screen-baseline`：保持阶段 0 的保存、发布、预览和冲突恢复行为不回退。
- Affected code（实施阶段的预期影响面，本次不修改）:
  - `apps/web/src/features/screen/components/screen-editor.tsx`：创建并下发编辑器会话交互控制器。
  - `apps/web/src/features/screen/components/screen-canvas.tsx`：按工具能力和交互状态启停选择、变换、平移、创建和缩放行为。
  - `apps/web/src/features/screen/components/canvas-context-menu.tsx`：同步右键菜单交互状态。
  - `apps/web/src/features/screen/components/canvas-status-bar.tsx`：消费统一工具注册表和真实活动工具。
  - `apps/web/src/features/screen/components/component-library.tsx`、图层面板及工具入口：保持创建和排序行为与统一会话状态协调。
  - `apps/web/src/features/screen/hooks/use-interaction-state-machine.ts`：补充生产事件、取消恢复和创建态。
  - `apps/web/src/features/screen/hooks/use-tool-state-machine.ts`：保留主工具与临时工具栈，移除重复文本编辑状态。
  - `apps/web/src/features/screen/hooks/use-modifier-keys.ts`、`use-keyboard-shortcuts.ts`、`shortcuts-registry.ts`：统一临时工具、快捷键和浏览器默认行为。
  - `apps/web/src/features/screen/lib/canvas-event-router.ts`：继续承载可纯函数测试的事件决策。
  - `apps/web/src/features/screen/stores/editor-store.ts`：只承载项目数据、选择、历史和视口等编辑状态，不承载 DOM 手势细节。
  - `apps/web/e2e/tests/`：新增编辑器画布交互与工具行为 E2E。

## ADDED Requirements

### Requirement: 单一编辑器会话交互控制器

系统 SHALL 通过一个会话级控制器组合工具状态、交互状态和文本编辑对象，并明确区分持久编辑状态、会话状态与单次 DOM 手势上下文。

#### Scenario: 编辑器建立单一会话状态源

- **WHEN** 大屏编辑器成功挂载
- **THEN** 编辑器只创建一个工具状态机实例和一个交互状态机实例
- **AND** 画布、工具入口、状态栏和快捷键读取同一实例的活动状态
- **AND** 不允许各子组件各自创建互不一致的活动工具或交互状态

#### Scenario: 高频手势数据保持局部

- **GIVEN** 用户正在拖拽、缩放、旋转、平移或绘制组件
- **WHEN** 系统记录 pointer 起点、DOM ref、Moveable 实例、实时辅助线或尺寸浮层
- **THEN** 这些数据保留在局部 ref、专用高频 Store 或手势会话中
- **AND** 不写入持久项目数据
- **AND** 不因统一状态源而导致每个 pointer move 重渲染整个编辑器

#### Scenario: 持久结果只通过编辑器 Store 提交

- **WHEN** 一次有效交互完成
- **THEN** 组件位置、尺寸、旋转、内容、样式、层级或新增组件通过现有编辑器 Store 动作提交
- **AND** 一次业务交互只产生约定数量的历史记录
- **AND** 取消或无变化交互不产生脏状态和空历史记录

### Requirement: 统一工具注册表

系统 SHALL 使用单一工具注册表定义选择、抓手、文字、矩形、椭圆、图片、缩放和吸管工具的元数据与能力。

#### Scenario: 多个入口展示一致

- **WHEN** 工具出现在工具入口、状态栏或快捷键帮助中
- **THEN** 名称、图标、`shortcutId`、光标和可用状态来自工具注册表
- **AND** 实际键位、scope、`preventDefault` 和浏览器冲突信息只来自 `SHORTCUTS_REGISTRY`
- **AND** 两个注册表通过 `shortcutId` 建立唯一引用，不重复保存实际键位字符串
- **AND** 无效引用、重复绑定和声明但未接入处理器的快捷键会被自动化测试拒绝

#### Scenario: 活动工具控制画布能力

- **WHEN** `activeTool` 发生变化
- **THEN** 画布依据工具能力决定是否允许 Selecto、Moveable 拖拽、缩放、旋转、平移、创建、采样或点击缩放
- **AND** 不允许工具只改变状态栏文字和 cursor 而不改变画布行为

#### Scenario: 临时工具可恢复

- **GIVEN** 用户的主工具不是抓手
- **WHEN** 用户按住 Space
- **THEN** 临时工具栈使活动工具变为抓手
- **AND** 松开 Space、窗口失焦或交互取消后恢复原主工具
- **AND** 不残留虚假的抓手状态

### Requirement: 交互状态机生产接入

系统 SHALL 将真实画布事件渐进接入统一交互状态机，并最终由状态机管理互斥交互状态。

#### Scenario: 镜像真实交互状态

- **WHEN** Selecto、Moveable、画布平移、锚点缩放、文本编辑或右键菜单开始和结束
- **THEN** 对应事件被派发到交互状态机
- **AND** 状态机状态与真实用户可观察行为一致
- **AND** 迁移初期不得仅因模型尚未覆盖的事件序列破坏现有行为

#### Scenario: 状态机成为仲裁源

- **GIVEN** 某类交互已经完成镜像验证和自动化覆盖
- **WHEN** 该交互迁移为生产仲裁
- **THEN** 是否允许冲突工具、Selecto、Moveable、右键菜单或快捷键由统一状态判断
- **AND** 删除决定同一语义的旧布尔状态
- **AND** 不一次性删除仍承载手势上下文的必要 ref

#### Scenario: 创建工具使用明确创建态

- **WHEN** 用户使用文字、矩形、椭圆或图片工具开始创建组件
- **THEN** 交互状态机进入明确的创建态并记录工具类型
- **AND** 创建态不复用框选态表达不同业务语义
- **AND** 完成或取消后回到可预测状态

#### Scenario: 统一取消和恢复

- **GIVEN** 当前处于框选、拖拽、缩放、旋转、平移、创建、缩放或文本编辑状态
- **WHEN** 发生 Escape、`pointercancel`、`lostpointercapture`、窗口失焦或组件卸载
- **THEN** 框选放弃未提交选择；拖拽、缩放和旋转恢复手势开始前几何；平移恢复开始前 offset；矩形和椭圆创建放弃预览；图片选择取消不创建组件；已完成缩放保留结果；文本 blur 或窗口失焦提交、Escape 回滚；右键菜单仅关闭
- **AND** `pointercancel`、`lostpointercapture` 和组件卸载对进行中的几何手势采用回滚语义，不提交中间 DOM 结果
- **AND** 执行回滚或放弃语义的取消路径不创建历史或错误脏状态；文本 blur 和窗口失焦属于 finalize，可在内容有效变化时创建一条历史
- **AND** 清理尺寸浮层、智能辅助线、pointer capture、创建预览和临时工具
- **AND** 状态机不会永久停留在瞬时交互状态

### Requirement: 选择与抓手工具闭环

系统 SHALL 让选择工具和抓手工具产生互斥且可预测的实际画布行为。

#### Scenario: 选择工具操作组件

- **GIVEN** 活动工具为选择工具
- **WHEN** 用户点击、框选或操作已选组件
- **THEN** Selecto 和适用的 Moveable 能力可用
- **AND** 用户可以选择、拖拽、缩放和旋转未锁定组件
- **AND** 空白点击按既有分组层级规则清理选择或退出活动分组

#### Scenario: 抓手工具平移画布

- **GIVEN** 活动工具为抓手工具
- **WHEN** 用户使用主指针拖动画布
- **THEN** 画布 offset 发生变化
- **AND** 组件项目坐标不发生变化
- **AND** Selecto 不开始框选，Moveable 不开始组件变换

#### Scenario: Space 临时抓手

- **GIVEN** 主工具为选择或创建工具且当前未处于禁止切换的文本输入阶段
- **WHEN** 用户按住 Space 并拖动画布
- **THEN** 行为与抓手工具一致
- **AND** 松开 Space 后恢复原主工具和对应光标

### Requirement: 文字工具与文本编辑闭环

系统 SHALL 为文字工具提供从创建到编辑退出的完整行为，并使用唯一文本编辑状态源。

#### Scenario: 点击创建文本

- **GIVEN** 活动工具为文字工具
- **WHEN** 用户点击允许创建的画布位置
- **THEN** 系统在对应画布坐标创建文本组件
- **AND** 新组件被选中并进入文本编辑状态
- **AND** 新组件在首次有效提交前属于临时创建会话
- **AND** 首次有效文本提交将创建与初始内容合并为一条可撤销历史

#### Scenario: 双击现有文本进入编辑

- **GIVEN** 选择工具处于可交互状态且目标为文本组件
- **WHEN** 用户双击该文本组件
- **THEN** 交互状态进入 `text-editing`
- **AND** 编辑对象 ID 指向该文本组件
- **AND** 双击文本不被误判为进入分组

#### Scenario: 文本编辑隔离画布快捷键

- **GIVEN** 用户正在编辑文本
- **WHEN** 用户输入字符、Space、方向键、Delete、Backspace 或系统复制粘贴快捷键
- **THEN** 文本输入优先获得这些事件
- **AND** 不触发删除组件、移动组件、临时抓手或其他画布命令
- **AND** 保存等明确允许的全局命令遵循既定策略

#### Scenario: 提交或取消文本编辑

- **GIVEN** 文本编辑器支持多行内容
- **WHEN** 用户按 Enter
- **THEN** 插入换行而不提交
- **WHEN** 用户按 Ctrl+Enter、Cmd+Enter 或使编辑器失焦
- **THEN** 有效文本写入组件并退出文本编辑状态
- **WHEN** 用户按 Escape
- **THEN** 回滚到进入编辑前内容并先退出文本编辑，不越级清空其他编辑器状态
- **AND** 新建文本在提交时为空白则放弃临时组件，不写入项目和历史
- **AND** 编辑已有文本的有效提交形成一条历史，无变化或取消编辑不创建历史
- **AND** 中文输入法 composition 期间 Enter 或 Escape 不得误提交、换行或取消

### Requirement: 阶段 1 组件类型与渲染闭环

系统 SHALL 为矩形、椭圆和图片建立共享 Schema 兼容的组件定义、默认值和 renderer，使工具创建结果同时可在编辑器与公开预览中渲染、保存和重新加载。

#### Scenario: 新组件类型完整注册

- **WHEN** 矩形、椭圆或图片组件被创建或从项目加载
- **THEN** registry 能识别组件类型并选择对应 renderer
- **AND** 编辑器与公开预览使用同一组件类型契约
- **AND** 未知组件占位不得被当作工具闭环完成

#### Scenario: 图片契约不足阻塞阶段完成

- **GIVEN** 现有项目契约无法安全持久化图片资源
- **WHEN** 图片资源审计完成
- **THEN** 阶段 1 被阻塞并创建补充提案
- **AND** 在安全契约落地前不得跳过图片工具并宣告阶段完成

### Requirement: 矩形与椭圆工具闭环

系统 SHALL 为矩形和椭圆工具提供拖拽创建行为。

#### Scenario: 拖拽创建形状

- **GIVEN** 活动工具为矩形或椭圆
- **WHEN** 用户在画布按下并拖拽到另一位置后释放
- **THEN** 系统以画布坐标创建对应形状组件
- **AND** 组件位置和尺寸与拖拽区域一致
- **AND** 创建过程显示直接操作预览
- **AND** 创建完成后新组件被选中

#### Scenario: 微小拖拽和取消

- **WHEN** 拖拽尺寸小于定义的最小创建阈值，或用户在完成前取消
- **THEN** 系统不创建不可用的零尺寸组件
- **AND** 不产生项目脏状态或空历史记录
- **AND** 创建预览被清理

#### Scenario: 形状默认样式一致

- **WHEN** 矩形或椭圆创建成功
- **THEN** 初始样式来自现有组件默认值或统一创建工厂
- **AND** 画布创建与组件库创建不会产生互相矛盾的默认 Schema

### Requirement: 图片工具闭环

系统 SHALL 允许用户通过图片工具创建可持久化的图片组件，并明确取消和失败行为。

#### Scenario: 选择图片并创建组件

- **GIVEN** 活动工具为图片工具
- **WHEN** 用户在画布指定位置并选择受支持的本地图片
- **THEN** 系统创建图片组件并显示可用预览
- **AND** 新组件位于指定位置并使用可预测的默认或原始宽高比
- **AND** 新组件被选中

#### Scenario: 取消图片选择

- **WHEN** 用户取消文件选择
- **THEN** 不创建组件
- **AND** 不写入历史或脏状态
- **AND** 工具和交互状态按既定策略恢复

#### Scenario: 不持久化本地绝对路径

- **WHEN** 图片组件写入项目数据
- **THEN** 项目不得保存用户设备的本地绝对文件路径
- **AND** 只保存应用允许的资源引用或数据形式
- **AND** 不在控制台或错误消息中泄露本地路径

### Requirement: 缩放工具闭环

系统 SHALL 让缩放工具以用户指针位置为锚点改变视口缩放。

#### Scenario: 点击锚点放大

- **GIVEN** 活动工具为缩放工具
- **WHEN** 用户点击画布
- **THEN** 画布围绕点击位置按定义步长放大
- **AND** 点击位置对应的画布内容在缩放前后保持视觉锚定
- **AND** 缩放值不超过既有上限

#### Scenario: 反向缩小

- **GIVEN** 活动工具为缩放工具
- **WHEN** 用户执行工具注册表定义的反向缩放操作
- **THEN** 画布围绕指针位置缩小
- **AND** 不触发选择、框选、组件拖拽或 Alt 拖拽复制
- **AND** 缩放值不低于既有下限

#### Scenario: 既有缩放入口保持一致

- **WHEN** 用户通过快捷键、修饰键滚轮、状态栏菜单或适应屏幕改变缩放
- **THEN** 所有入口使用同一缩放边界与视口更新语义
- **AND** 浏览器页面缩放或页面滚动不会覆盖已声明由编辑器处理的组合键

### Requirement: 吸管工具最小闭环（已移出阶段 1 范围）

> **2026-07-19 决策**：吸管工具已移出阶段 1 范围。
> 原因：阶段 1 未设计调色板或活动颜色消费 UI，吸管的"采样到活动颜色"半边无可观察结果，
> 而"应用到选中组件"半边与"切换工具清空选中"约束直接冲突（切到吸管即清空选中，应用目标为空）。
> 按"不宣称无效能力""工具必须产生可观察画布行为"原则，保留吸管反而违反 Spec。
> 处理方式：彻底删除吸管相关代码（color-sampler.ts、TOOL_REGISTRY 条目、sampling 状态、
> activeColor 字段、状态栏颜色显示、吸管 E2E），待后续阶段设计调色板时重新引入。
> 以下原始 Scenario 保留作为历史记录，当前不生效。

系统 SHALL 为吸管工具提供安全、可测试的编辑器内颜色采样和应用行为。

#### Scenario: 采样编辑器拥有的可见颜色

- **GIVEN** 活动工具为吸管且指针位于编辑器拥有的组件或画布表面
- **WHEN** 用户点击一个可解析的纯色区域
- **THEN** 系统将解析出的规范颜色写入会话活动颜色
- **AND** 状态栏或工具反馈显示采样成功
- **AND** 不依赖跨域 canvas 像素读取

#### Scenario: 应用到支持颜色的选中组件

- **GIVEN** 已选择一个支持目标颜色属性的组件
- **WHEN** 吸管成功采样颜色
- **THEN** 该颜色通过现有 Store 更新动作应用到明确的目标颜色属性
- **AND** 该应用可撤销
- **AND** 不支持颜色的组件不会被写入无效 Schema 字段

#### Scenario: 不可采样目标安全失败

- **WHEN** 用户点击跨域图片像素、透明区域或无法解析颜色的目标
- **THEN** 系统不伪造颜色值
- **AND** 不修改组件和历史
- **AND** 给出非阻塞反馈并保持编辑器可继续操作

### Requirement: 工具入口与反馈一致

系统 SHALL 提供可访问的工具选择入口，并确保界面反馈与真实画布模式一致。

#### Scenario: 工具快捷键真实绑定

- **WHEN** 工具注册表通过 `shortcutId` 声明快捷键
- **THEN** 对应 `SHORTCUTS_REGISTRY` 条目和生产处理器真实存在
- **AND** 文本编辑或表单输入期间不会错误切换工具
- **AND** 没有快捷键的工具显式使用空引用，而不是伪造未绑定按键

#### Scenario: 用户可通过界面选择工具

- **WHEN** 用户使用鼠标或键盘导航工具入口
- **THEN** 可以选择阶段 1 范围内的工具
- **AND** 当前工具具有可访问名称和选中状态
- **AND** 键盘焦点可见且不会被画布全局快捷键错误吞掉

#### Scenario: 工具反馈一致

- **WHEN** 主工具或临时工具变化
- **THEN** 工具入口、状态栏、光标和画布行为显示相同的活动工具
- **AND** 临时工具结束后四者共同恢复

#### Scenario: 不宣称无效能力

- **WHEN** 某个开关、工具或快捷键尚无生产行为
- **THEN** 界面不得将其呈现为已完成且可工作的能力
- **AND** 快捷键帮助不得描述未绑定或语义冲突的操作

### Requirement: 关键组合交互浏览器回归

系统 SHALL 使用 Playwright 覆盖仅靠纯函数或 jsdom 无法证明的关键编辑器组合交互。

#### Scenario: 框选后拖拽

- **GIVEN** 画布存在多个可选择组件
- **WHEN** 用户框选组件并拖拽已选集合
- **THEN** 选中集合按同一位移更新
- **AND** 未选组件不移动
- **AND** 操作结束后状态回到可继续选择的状态

#### Scenario: 双击进入分组

- **GIVEN** 画布存在分组组件
- **WHEN** 用户按既有交互规则双击分组或分组内组件
- **THEN** 活动分组和选择结果正确更新
- **AND** 双击判定不与框选结束或文本编辑冲突

#### Scenario: Alt 拖拽复制

- **GIVEN** 活动工具为选择工具且组件可拖拽
- **WHEN** 用户按住 Alt 拖拽组件并释放
- **THEN** 原组件保持原位并产生位于目标位置的副本
- **AND** 该操作形成可撤销的单一业务历史
- **AND** 浏览器默认行为不抢占该组合操作

#### Scenario: 图层拖拽排序

- **WHEN** 用户在图层面板拖拽组件改变顺序
- **THEN** 组件层级按目标位置更新
- **AND** 画布视觉层级同步变化
- **AND** 画布交互状态不会误进入组件拖拽

#### Scenario: 快捷键缩放

- **WHEN** 用户使用编辑器声明的放大、缩小或适应屏幕快捷键
- **THEN** 编辑器视口按约定变化
- **AND** 浏览器页面缩放、页面查找、历史导航或滚动不会覆盖应用行为

#### Scenario: 右键菜单组合交互

- **WHEN** 用户分别右键组件和画布空白、在菜单打开时再次右键并关闭菜单
- **THEN** 菜单类型和位置正确更新
- **AND** 关闭后仍可继续选择和拖拽组件
- **AND** 菜单不会与文本编辑或创建态同时进入冲突状态

#### Scenario: 测试独立与可恢复

- **WHEN** 编辑器交互 E2E 执行
- **THEN** 每个用例独立创建并清理所需项目数据
- **AND** 不依赖用例顺序、残留登录态或固定项目 ID
- **AND** 失败时保留足以定位工具、状态和 DOM 行为的证据

## MODIFIED Requirements

### Requirement: 既有交互模块从研究骨架升级为生产能力

`research-interaction-architecture` 中已完成的状态机、事件路由、工具栈、修饰键和反馈层 SHALL 保持其可测试边界，但阶段 1 完成定义由“模块存在”修改为“真实生产路径接入并有浏览器级证据”。

#### Scenario: 模块存在不等于能力完成

- **GIVEN** 某个 Hook、纯函数或状态枚举已经存在
- **WHEN** 它未被真实编辑器事件调用，或调用后没有可观察结果
- **THEN** 该能力不得标记为阶段 1 完成
- **AND** 必须由集成测试或 E2E 证明生产接入

### Requirement: 现有画布基础能力保持兼容

系统 SHALL 在引入统一状态源时保留现有 Moveable、Selecto、分组、辅助线、吸附、历史和视口能力，除非本 Spec 明确修改其交互语义。

#### Scenario: 渐进迁移不重写底层库

- **WHEN** 交互状态机接入选择、拖拽、缩放、旋转和平移
- **THEN** 继续复用现有 Moveable、Selecto 和事件路由能力
- **AND** 不因架构统一而替换成熟底层库或重写全部画布

#### Scenario: 阶段 0 行为不回退

- **WHEN** 阶段 1 修改组件、Store 或快捷键链路
- **THEN** 保存、发布、公开预览、冲突恢复、属性同步和编辑预览样式一致性继续通过既有测试

## REMOVED Requirements

### Requirement: 删除重复的文本编辑状态源

系统 SHALL 删除工具状态机中与交互状态机重复表达“正在编辑文本”的独立布尔状态，由唯一会话状态和编辑对象 ID 表达文本编辑。

#### Scenario: 快捷键只读取唯一文本编辑状态

- **WHEN** 快捷键判断是否应让位于文本输入
- **THEN** 只读取统一交互会话状态
- **AND** 不再读取另一份可能不同步的 `isEditingText`

### Requirement: 删除决定同一交互的旧布尔状态

系统 SHALL 在每个交互场景完成状态机接入和回归验证后，删除决定同一交互语义的旧布尔状态；承载手势数据的必要 ref 不属于本删除范围。

#### Scenario: 平移状态迁移完成

- **GIVEN** 抓手和平移已经由活动工具与交互状态机控制
- **WHEN** 旧 `isPanning` 仅用于重复表达当前交互状态
- **THEN** 删除该重复状态或改为从统一状态派生
- **AND** pointer 起点和原 offset 等手势上下文继续保留在局部 ref

### Requirement: 删除错误或无行为的工具说明

系统 SHALL 删除或修正与真实绑定不一致的快捷键和工具文案。

#### Scenario: 吸管快捷键说明与真实行为一致

- **WHEN** 快捷键帮助展示吸管或 Alt 相关操作
- **THEN** 文案与实际工具绑定、Alt 拖拽复制和缩放工具反向操作一致
- **AND** 不再宣称未实现的“Alt 临时吸管”行为

## Out of Scope

- 动态 API 数据源、数据映射编辑器及“数据、逻辑、视觉、交互”四层配置。
- 服务端修订版（revision）、结构化 diff、版本恢复和发布快照。
- 项目成员权限、角色工作区、多人协作、Presence、评论和 WebSocket。
- CRDT、OT、分支合并和 AI 辅助。
- 以状态机接入为名重写 Moveable、Selecto、组件 Schema 或整个编辑器 Store。
- 跨域图片像素级吸管采样、屏幕级取色器或需要额外原生权限的系统取色能力。
- 图片服务端素材库、上传管理后台和资源 CDN 平台；阶段 1 只完成现有项目契约允许的本地图片创建最小闭环。
- 为了阶段 1 扩大后端 API 或数据库模型；若实现审计发现确需后端能力，必须停止对应子任务并另行提案。
