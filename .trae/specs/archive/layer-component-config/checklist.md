# Checklist

本 checklist 用于验收 `layer-component-config` 阶段 2 实现。所有实施条目默认未完成；只有产品行为、自动化验证和必要浏览器证据均存在时才能勾选。

## 规格边界

- [x] 实施范围只覆盖四层配置契约、数据解析器、bar-chart 数据源闭环、数据源配置 UI、三态、定时刷新、画布配置历史和敏感请求头保护。
- [x] 未引入 POST、PUT、DELETE 等写请求方法或请求体编辑。
- [x] 未引入 WebSocket 或其他数据源类型。
- [x] 未引入 ECharts、Recharts 等图表库；bar-chart 仍为手写 SVG renderer。
- [x] 未引入多系列、聚合、分组、下钻或图表联动能力。
- [x] 未将数据源接入扩展到 bar-chart 以外的组件类型。
- [x] 未建设服务端 API 代理、跨域转发或凭证托管。
- [x] 未引入全局共享数据源管理。
- [x] 未引入服务端修订版（revision）、项目分支、实时多人同步或 AI 推荐。
- [x] 未引入角色工作区或按角色的面板裁剪。
- [x] 未为 bar-chart 以外的组件类型建立 props 强类型契约。
- [x] 未一次性重写属性面板或编辑器 Store。

## 四层配置契约

- [x] 数据层配置只存在于 `dataSource` 字段。
- [x] 数据层包含数据源类型、静态数据、API 配置、可选数据路径与可选字段映射。
- [x] 逻辑层配置只存在于 `logic` 字段，包含排序与条数限制。
- [x] 视觉层配置只存在于 `style` 与视觉类 props。
- [x] 交互层配置只存在于 `interaction` 字段。
- [x] `position`、`zIndex`、`status`、`parentId` 未被混入四层配置。
- [x] 每一层配置可单独通过共享 Zod Schema 校验。
- [x] 单层校验错误不掩盖其他层校验结果。
- [x] 校验错误信息可定位到具体层与字段。
- [x] 修改任一层时其余三层配置值不变。
- [x] 服务端保存接口使用同一共享 Schema 校验四层配置。
- [x] 非法四层配置不会写入数据库。

## 数据解析器

- [x] 解析器为纯函数，不发起 IO、不产生副作用。
- [x] 解析管线依次执行数据路径提取、字段映射、逻辑层处理。
- [x] 输出为规范化图表数据（`{ name, value }` 条目列表）。
- [x] 嵌套数据路径可正确提取目标数组。
- [x] 数据路径不存在返回结构化错误。
- [x] 非数组输入返回结构化错误。
- [x] 映射字段缺失返回结构化错误。
- [x] 非数值映射值返回结构化错误。
- [x] 各类错误原因可区分。
- [x] 解析器不抛出未捕获异常。
- [x] 错误信息可读且不泄露原始数据全文。
- [x] 合法空数组返回空结果而非错误。
- [x] 未配置字段映射时按 `name`→维度、`value`→数值默认推断。
- [x] 排序作用于映射结果副本，原始数据引用不变。
- [x] 条数限制正确截断。
- [x] 空逻辑配置透传映射结果。
- [x] 静态与 API 响应复用同一解析管线入口。

## API 数据源绑定
