# Checklist

本 checklist 用于验收 `stabilize-screen-baseline` 阶段实现。所有条目默认未完成；只有实现、自动化验证和必要手动证据均存在时才能勾选。

## 规格边界

- [x] 实施范围只覆盖预览隔离、发布边界、认证路由、共享样式、属性同步、`expectedUpdatedAt` 乐观锁、冲突 UI 和测试基线。
- [x] 未引入项目所有者/编辑者/查看者成员权限矩阵。
- [x] 未引入服务端修订版（revision）、自动合并、CRDT、OT 或实时协作。
- [x] 未把发布版本快照或草稿/线上双版本模型混入阶段 0。
- [x] 未夹带与阶段 0 无关的大规模编辑器重构、新工具或新数据源功能。

## 公开预览隔离

- [x] 公开预览控制器调用专用公开查询服务，而不是普通受保护详情服务。
- [x] 公开查询的数据条件同时包含项目 ID 和 `published` 状态。
- [x] 已发布项目可在无访问令牌时通过公开预览接口读取。
- [x] 草稿项目通过公开预览接口返回未找到类错误。
- [x] 草稿失败响应不包含项目名称、画布、组件或缩略图数据。
- [x] 不存在项目与草稿项目在公开预览中均不泄露项目内容。
- [x] 公开预览页面对草稿和不存在项目展示统一的“不存在或未发布”状态。
- [x] 公开预览页面只调用公开预览接口，不调用受保护项目详情接口。
- [x] 匿名预览能力不允许创建、更新、发布、删除或受保护详情读取。

## 认证路由

- [x] 大屏列表路由位于受保护认证边界内。
- [x] 大屏编辑器路由在编辑器挂载和数据请求前执行认证判断。
- [x] 未认证直达大屏列表会重定向到登录页。
- [x] 未认证直达 `/screen/:id` 会重定向到登录页。
- [x] 未认证重定向过程中不闪现项目名称、画布或组件内容。
- [x] 公开预览路由不位于受保护布局下。
- [x] 未认证用户可直接打开已发布项目的公开预览路由。
- [x] 未认证直接调用列表、详情、创建、更新、发布和删除接口均返回 401。
- [x] 服务端认证仍是最终安全边界，前端路由保护不是唯一防线。

## 保存与发布边界

- [x] 普通保存只更新名称、描述、画布、组件和缩略图等可编辑字段。
- [x] 普通保存由服务端将项目状态设为 `draft`。
- [x] 已发布项目保存成功后立即退出公开可见状态。
- [x] 已发布项目保存后必须再次发布，公开预览才可读取新保存内容。
- [x] 保存请求携带 `expectedUpdatedAt`，其值为客户端最后确认的服务端 `updatedAt`。
- [x] 保存成功响应返回完整项目和新的 `updatedAt`。
- [x] 保存成功后编辑器 Store 使用服务端响应回写，而不是自行生成新时间。
- [x] 保存成功后详情缓存和列表缓存与服务端状态一致。
- [x] 编辑器能够判断相对最后加载/保存响应的本地脏状态。
- [x] 加载项目后本地状态为干净。
- [x] 修改画布或组件后本地状态为脏。
- [x] 保存成功后本地状态恢复干净。
- [x] 保存失败或冲突后本地状态保持为脏。
- [x] 存在未保存修改时发布请求不会发送。
- [x] 存在未保存修改时用户得到明确的先保存提示。
- [x] 发布请求只包含项目 ID 所在路径与 `expectedUpdatedAt`，不携带画布或组件内容。
- [x] 发布成功只发布服务端当前已保存内容。
- [x] 发布成功返回 `published` 状态和新的 `updatedAt`。
- [x] 发布成功后编辑器 Store、详情缓存和列表缓存均回写新状态与新基线。
- [x] 发布成功后当前项目的公开预览缓存被失效或更新。
- [x] 过期基线发布不会改变项目状态。
- [x] 发布失败或冲突不会显示“发布成功”。

## 共享组件样式

- [x] 存在单一纯函数负责组件公共容器样式解析。
- [x] 共享样式解析函数不依赖编辑器 Store、DOM、window、Moveable 或 React 组件状态。
- [x] 共享样式覆盖 `left`、`top`、`width`、`height` 和 `zIndex`。
- [x] 共享样式覆盖 `opacity`、`backgroundColor`、`borderRadius` 和 `overflow`。
- [x] 共享样式覆盖 `borderWidth`、`borderColor` 和 `borderStyle`。
- [x] 共享样式覆盖非零 `position.rotation` 的 `transform`。
- [x] 编辑器组件容器使用共享样式解析结果。
- [x] 预览组件容器使用同一共享样式解析结果。
- [x] 预览中旋转角度与编辑器一致。
- [x] 画布整体缩放不覆盖组件自身旋转。
- [x] 隐藏组件不在公开预览中可见。
- [x] 编辑器选中框、辅助线和 Moveable 控件未进入共享样式。
- [x] 公开预览不依赖编辑器 Store 或交互组件。
- [x] 删除预览旋转接入会导致自动化测试失败。

## 属性同步

- [x] 属性面板以编辑器 Store 当前组件数据为权威值。
- [x] 未聚焦输入框时，画布拖拽完成后 X/Y 显示最新值。
- [x] 未聚焦输入框时，画布缩放完成后宽/高显示最新值。
- [x] 未聚焦输入框时，画布旋转完成后旋转值显示最新值。
- [x] 用户编辑草稿时，无关重渲染不会清空草稿。
- [x] 用户编辑组件 A 时切换到组件 B，不会把 A 的草稿提交到 B。
- [x] 同一属性被画布外部更新后，旧草稿不能覆盖新权威值。
- [x] Escape 放弃草稿并恢复权威值。
- [x] Enter 提交一次有效更新。
- [x] Enter 引发的 Blur 不会重复提交。
- [x] Blur 提交一次有效更新。
- [x] ArrowUp/ArrowDown 按已定义步长提交并阻止浏览器默认行为。
- [x] Shift+ArrowUp/ArrowDown 使用已定义的大步长。
- [x] 空白、无效和未变化输入不写入 Store。
- [x] 一次属性提交只产生一次业务更新和预期历史记录。

## expectedUpdatedAt 乐观锁契约

- [x] 共享错误定义包含专用的大屏保存冲突业务码。
- [x] 保存冲突业务码映射为 HTTP 409。
- [x] 更新请求 Schema 将 `expectedUpdatedAt` 定义为必填保存基线。
- [x] 发布请求使用独立 Schema，且 `expectedUpdatedAt` 为必填。
- [x] 请求 `expectedUpdatedAt` 与响应 `updatedAt` 均使用现有日期时间字符串 Schema。
- [x] 服务端不信任客户端传入的 `expectedUpdatedAt` 作为写入后的目标时间。
- [x] 更新通过项目 ID 且数据库 `updatedAt` 等于请求 `expectedUpdatedAt` 的单次条件写入执行。
- [x] 发布通过项目 ID 且数据库 `updatedAt` 等于请求 `expectedUpdatedAt` 的单次条件写入执行。
- [x] 基线匹配时更新成功且受影响记录数为一。
- [x] 基线匹配时发布成功且受影响记录数为一。
- [x] 基线过期时更新不修改任何记录。
- [x] 基线过期时发布不修改任何记录。
- [x] 更新条件未命中时可区分项目不存在与保存冲突。
- [x] 发布条件未命中时可区分项目不存在与保存冲突。
- [x] 项目不存在返回项目未找到业务错误。
- [x] 基线过期返回专用保存冲突业务错误。
- [x] 服务端不执行最后写入覆盖。
- [x] 服务端不在阶段 0 自动合并冲突内容。
- [x] 客户端保存成功后将响应中的新 `updatedAt` 作为下一次保存请求的 `expectedUpdatedAt`。
- [x] 客户端发布成功后将响应中的新 `updatedAt` 作为后续请求的 `expectedUpdatedAt`。

## 冲突 UI

- [x] 前端通过专用业务码识别保存和发布冲突。
- [x] 冲突识别不依赖中文或英文错误消息文本。
- [x] 保存冲突会打开阻塞式对话框，而不只是显示 Toast。
- [x] 发布冲突复用同一冲突处理模型。
- [x] 对话框有明确可访问标题和冲突原因说明。
- [x] 对话框说明当前本地修改尚未保存。
- [x] 对话框说明重新加载会放弃本地未保存修改。
- [x] 对话框提供取消/留在当前编辑器操作。
- [x] 对话框提供重新加载服务端版本操作。
- [x] 冲突发生后本地编辑器 Store 内容保持不变。
- [x] 冲突发生后本地历史保持可用，直至用户确认重新加载。
- [x] 冲突发生后旧保存基线不被伪装成最新值。
- [x] 用户取消后本地内容保持不变。
- [x] 用户取消后再次保存仍使用旧基线。
- [x] 用户确认重新加载后重新请求受保护项目详情。
- [x] 重新加载成功后完整替换项目数据和保存基线。
- [x] 重新加载成功后选中态被安全重置。
- [x] 重新加载成功后属性输入草稿被清理。
- [x] 重新加载成功后未提交本地历史按约定重置。
- [x] 重新加载失败时本地内容不丢失。
- [x] 重新加载失败时用户可重试或取消。
