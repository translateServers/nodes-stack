# Checklist

本 checklist 验证整体规划是否足以指导后续按阶段编写独立 Spec，不用于验证尚未开始的功能实现。

## 规划结构

- [x] `spec.md` 包含 Why、What Changes、Impact、ADDED、MODIFIED、REMOVED Requirements。
- [x] `tasks.md` 明确说明本文件只做阶段规划，不直接一次性实施全部功能。
- [x] 整体规划使用小步快跑原则，每个阶段均可独立交付和验收。
- [x] 规划明确要求每个进入实施的阶段另建独立 `spec.md`、`tasks.md` 和 `checklist.md`。
- [x] 规划没有在 Proposal 阶段修改产品代码。

## 阶段完整性

- [x] 阶段 0「可靠性与事实基线」覆盖公开预览发布状态隔离、受保护项目接口身份认证、编辑预览一致性、属性面板同步、基于保存基线的乐观锁和测试基线。
- [x] 阶段 1「单用户交互闭环」覆盖交互状态机生产接入、工具行为闭环和编辑器关键 E2E。
- [x] 阶段 2「非破坏性数据与交互分层」覆盖“数据、逻辑、视觉、交互”四层契约、静态数据源和 API 数据源最小闭环。
- [x] 阶段 3「服务端修订版与恢复」覆盖服务端修订版（revision）、结构化 diff、安全恢复和本地项目快照兼容。
- [x] 阶段 4「权限隔离与角色工作区」覆盖全局权限、项目成员权限、前端权限裁剪和角色工作区，并区分成员角色与工作区类型。
- [x] 阶段 5「实时协作基础」覆盖 WebSocket 项目房间、Presence、增量同步、重连补偿和评论批注。
- [x] 阶段 6「分支合并与智能辅助评估」将复杂分支合并和 AI 辅助定义为后置评估，而不是提前承诺实施。

## 小步快跑边界

- [x] 每个阶段都有明确的目标和用户价值。
- [x] 每个阶段都有明确范围和“明确不做”的内容。
- [x] 每个阶段都有进入条件和退出条件。
- [x] 阶段依赖顺序合理：阶段 0 建立身份认证基线，阶段 3 记录操作者，阶段 4 建立项目成员授权，阶段 5 才进入实时协作。
- [x] 阶段 3 不依赖阶段 4 的项目成员角色矩阵，阶段 4 也不反向阻塞阶段 3 的基础操作者审计。
- [x] 首个待细化阶段明确为阶段 0。
- [x] 阶段 0 提供了可继续拆解的能力链，但未直接展开为耦合的超大实现任务。
- [x] 规划明确共享 Schema 应先稳定，再接入前后端。
- [x] 规划明确数据模型、接口、UI、迁移和测试应分开交付。

## 研究审计对齐

- [x] 规划区分“模块已实现”与“生产路径已接入”。
- [x] 规划统一使用“本地编辑历史”“本地项目快照”“保存基线”“服务端修订版（revision）”，且未混淆这些概念。
- [x] 规划承认现有基础画布交互已完成较多，不重复推翻已有架构。
- [x] 规划覆盖审计发现的公开预览草稿泄露风险。
- [x] 规划覆盖编辑与预览 rotation/公共样式不一致问题。
- [x] 规划覆盖属性面板局部状态可能过期问题。
- [x] 规划覆盖并发保存最后写入覆盖问题。
- [x] 规划覆盖交互状态机未接入生产画布问题。
- [x] 规划覆盖工具只有状态、没有业务行为的问题。
- [x] 规划覆盖数据源只有 Schema、没有运行时链路的问题。
- [x] 规划覆盖后端 RBAC、项目成员权限和角色工作区缺失问题。
- [x] 规划覆盖实时协作、评论和服务端修订版差异缺失问题。

## 安全与兼容性

- [x] API 数据源要求敏感请求头不暴露在公开预览数据中。
- [x] 公开预览明确只允许读取已发布项目，且不授予任何编辑或项目成员权限。
- [x] 未认证访问受保护接口返回 401，已认证但权限不足返回 403，非成员资源探测返回不泄露存在性的 404。
- [x] 权限要求由后端执行，不能仅依赖前端隐藏。
- [x] 项目成员角色统一为所有者/编辑者/查看者，开发者/业务员仅为工作区类型且不得扩大权限。
- [x] AI 建议不得绕过权限、服务端修订版和审计自动覆盖项目。
- [x] 后续数据迁移要求提供兼容读取或迁移方案。
- [x] 本地项目快照在服务端修订版上线后仍可读取或导入。

## 验证策略

- [x] 每个阶段完成前要求通过独立 checklist。
- [x] 规划要求按阶段运行适用的 typecheck、lint、Biome、单元测试或 E2E。
- [x] 关键组合交互要求浏览器级 E2E，而非仅依赖纯函数测试。
- [x] 实时协作阶段要求双浏览器验证、断线重连验证、未认证连接验证和无项目权限连接验证。
- [x] 规划要求重新核实当前测试基线，旧测试数字不得继续作为当前完成证据。

## 文档质量

- [x] 三份文档均使用中文。
- [x] 阶段编号、名称和顺序在三份文档中一致。
- [x] 三份文档统一使用“数据、逻辑、视觉、交互”四层配置、服务端修订版（revision）、本地项目快照等术语。
- [x] Requirement 使用 SHALL，并包含可验证 Scenario。
- [x] 没有将实现细节过早锁定为不必要的复杂技术方案。
- [x] 没有创建 `spec.md`、`tasks.md`、`checklist.md` 之外的文档。
