# Tasks

## 执行原则

- 本文件只拆分阶段 1 实施任务，不代表产品代码已经开始或任务已经完成。
- 阶段 0 checklist、验证记录和关键回归全部有效是强制门禁；门禁失效时不得开始或继续阶段 1。
- 严格采用“状态定义 → 单场景镜像接入 → 自动化验证 → 切换仲裁 → 删除对应旧状态”的顺序。
- 每个任务只交付一个可观察、可验证、可回滚的结果；不得一次性重写 `ScreenCanvas`。
- 保留 Moveable、Selecto、事件路由纯函数、尺寸浮层和智能辅助线高频 Store，除非任务明确要求调整边界。
- 工具注册、画布行为、快捷键、反馈和测试分开交付，但工具没有可观察行为和回归测试时不得标记完成。
- 取消、无变化和失败操作不得产生空历史记录或错误脏状态。
- 不在阶段 1 引入动态 API 数据源、服务端修订版、权限矩阵、多人协作或后端资源平台。
- 任务状态仅在实现、自动化验证和必要浏览器证据真实完成后从 `[ ]` 改为 `[x]`。
